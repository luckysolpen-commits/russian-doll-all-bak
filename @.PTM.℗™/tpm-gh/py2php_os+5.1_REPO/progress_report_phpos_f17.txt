## Technical Implementation

### Key Breakthrough: Dual Format Persistence

```javascript
// piece-manager.js
async _savePieceToPHP(pieceId, piece) {
    // Save piece.json (full definition)
    await fetch(`${phpApi}?action=write`, {
        method: 'POST',
        body: JSON.stringify({
            path: `pieces/${pieceId}/piece.json`,
            content: JSON.stringify(piece, null, 2),
            format: 'json'
        })
    });
    
    // ALSO save state.txt (Python TPM key-value format)
    const stateLines = [];
    for (const [key, value] of Object.entries(piece.state)) {
        stateLines.push(`${key} ${value}`);
    }
    const stateContent = stateLines.join('\n') + '\n';
    
    await fetch(`${phpApi}?action=write`, {
        method: 'POST',
        body: JSON.stringify({
            path: `pieces/${pieceId}/state.txt`,
            content: stateContent,
            format: 'keyvalue'
        })
    });
}
```

### Load Priority Chain

```javascript
async loadPiece(pieceId) {
    // 1. Try state.txt first (Python TPM pattern)
    const state = await fetch(`pieces/${pieceId}/state.txt`);
    if (state.ok) return parseKeyValueState(state);
    
    // 2. Fallback to piece.json
    const piece = await fetch(`pieces/${pieceId}/piece.json`);
    if (piece.ok) return piece.json();
    
    // 3. Fallback to localStorage
    const saved = localStorage.getItem(`piece_${pieceId}`);
    if (saved) return JSON.parse(saved);
    
    // 4. Use default
    return createDefaultPiece(pieceId);
}
```

## Known Issues / Limitations

### Current Limitations
1. **No multi-user support** - Single session only
2. **No save slots** - Only one state per piece
3. **No piece-to-piece communication** - Direct method calls instead of ledger events
4. **No master ledger monitor thread** - JavaScript single-threaded
5. **No plugin hot-reloading** - Must refresh browser

### Browser-Specific Issues
1. **localStorage quota** - 5-10MB limit per domain
2. **PHP required** - Won't work as pure static HTML
3. **CORS** - Must run on PHP server (not file://)

| Feature | Python TPM | Web TPM | Notes |
|---------|-----------|---------|-------|
| State format | key-value .txt | key-value .txt | ✅ Identical |
| Piece format | piece.txt | piece.json | ✅ Equivalent |
| Save trigger | Every change | Every change | ✅ Identical |
| Load source | state.txt | state.txt | ✅ Identical |
| Master ledger | master_ledger.txt | ledger.json | ✅ Equivalent |
| Plugin system | .py modules | .js modules | ✅ Equivalent |
| Event system | File-based | In-memory + files | ⚠️ Hybrid |
