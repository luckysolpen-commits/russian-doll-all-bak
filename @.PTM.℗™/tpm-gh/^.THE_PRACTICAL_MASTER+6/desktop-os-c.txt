# DESKTOP OS in GCC C - Development Plan

## Overall Goal
Develop a basic "DESKTOP OS" in C using GCC, demonstrating core operating system concepts, similar to previous GCC projects.

## Phase 1: Setup and Basic Environment
- [ ] **Subtask 1.1**: Set up cross-compilation toolchain for a target architecture (e.g., i386 or x86_64). This involves installing necessary compilers and utilities.
- [ ] **Subtask 1.2**: Create a minimal bootloader (assembly or C with inline assembly) to initialize the CPU and load the kernel from disk.
- [ ] **Subtask 1.3**: Implement a basic kernel entry point (`kernel_main`) and print "Hello, OS!" to the console using direct VGA text mode access or a simple serial driver.
- [ ] **Subtask 1.4**: Set up a `Makefile` for automating the build process, including compiling the bootloader and kernel, and creating an ISO image suitable for booting in a virtual machine.

## Phase 2: Essential Kernel Services
- [ ] **Subtask 2.1**: Implement a Global Descriptor Table (GDT) for memory segmentation to define different memory regions and access privileges.
- [ ] **Subtask 2.2**: Set up an Interrupt Descriptor Table (IDT) and basic interrupt handlers for CPU exceptions (e.g., division by zero, page fault) and hardware interrupts (e.g., programmable interrupt controller - PIC).
- [ ] **Subtask 2.3**: Implement a basic timer interrupt handler to provide system ticks.
- [ ] **Subtask 2.4**: Implement a basic keyboard driver to handle keyboard interrupts and read input from the keyboard.
- [ ] **Subtask 2.5**: Set up a simple screen driver (VGA text mode) for robust text output, including scrolling and cursor management.

## Phase 3: Memory Management
- [ ] **Subtask 3.1**: Implement a physical memory manager (PMM) (e.g., using a bitmap or linked list) to track and allocate/deallocate physical memory pages.
- [ ] **Subtask 3.2**: Implement a virtual memory manager (VMM) using paging, setting up page directories and page tables to map virtual addresses to physical addresses.
- [ ] **Subtask 3.3**: Implement `kmalloc`/`kfree` (kernel heap allocator) for dynamic memory allocation within the kernel.

## Phase 4: Process Management
- [ ] **Subtask 4.1**: Define a Process Control Block (PCB) structure to store process-specific information (registers, stack pointer, memory maps, etc.).
- [ ] **Subtask 4.2**: Implement a basic process creation mechanism (`fork`-like functionality) to create new processes.
- [ ] **Subtask 4.3**: Implement a simple scheduler (e.g., round-robin) to manage process execution and context switching.
- [ ] **Subtask 4.4**: Implement basic process destruction (`exit`-like functionality) to clean up resources.

## Phase 5: File System (Basic)
- [ ] **Subtask 5.1**: Implement a simple virtual file system (VFS) interface.
- [ ] **Subtask 5.2**: Implement a ramdisk-based file system or a very basic FAT12/FAT16 driver to load files from the boot medium.
- [ ] **Subtask 5.3**: Implement basic file operations: open, close, read, write, seek, list directory.

## Phase 6: User Interface & Applications
- [ ] **Subtask 6.1**: Implement a simple command-line interface (shell) that can parse user input and execute built-in commands (e.g., `help`, `echo`, `clear`, `ls`, `cat`).
- [ ] **Subtask 6.2**: Implement basic system calls for user programs to interact with the kernel (e.g., for I/O, process control).
- [ ] **Subtask 6.3**: (Optional, advanced) Start working on a very basic graphical output using VGA graphics modes (e.g., pixel drawing, simple shapes).

## Phase 7: Testing and Debugging
- [ ] **Subtask 7.1**: Set up QEMU as the primary emulator for running and testing the OS.
- [ ] **Subtask 7.2**: Configure GDB for remote kernel debugging with QEMU.
- [ ] **Subtask 7.3**: Implement basic logging mechanisms (e.g., serial port output) for kernel debugging.
