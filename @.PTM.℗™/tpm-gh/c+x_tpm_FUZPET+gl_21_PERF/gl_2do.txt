# GL Renderer Implementation Plan
# Following TPM standards - everything is a piece

## Overview
Add a GL-based renderer that displays the same content as the terminal renderer,
but using OpenGL with the Noto Color Emoji font for rendering.

## Architecture

### Current Flow (Terminal)
1. keyboard_input.c → reads stdin → writes to pieces/keyboard/history.txt
2. game_logic.c → reads keyboard/history.txt → updates game state → render_print() → writes to pieces/display/current_frame.txt
3. renderer.c → reads pieces/display/current_frame.txt → prints to terminal

### New Flow (GL)
1. keyboard_input.c → reads stdin → writes to pieces/keyboard/history.txt (unchanged)
2. game_logic.c → reads keyboard/history.txt → updates game state → render_print() + gl_render_print() → writes to:
   - pieces/display/current_frame.txt (for terminal renderer)
   - pieces/gl_display/current_frame.txt (for GL renderer)
3. gl_renderer.c → reads pieces/gl_display/current_frame.txt → renders with GL/emoji font

## Tasks

### 1. Create gl_display piece directory structure
   pieces/gl_display/
   ├── piece.txt          # Piece definition
   ├── history.txt        # Input history
   ├── ledger.txt         # Audit trail
   └── current_frame.txt  # Current frame data (written by game_logic)

### 2. Modify game_logic.c
   - Add gl_render_print() function (mirrors render_print())
   - Call gl_render_print() alongside render_print() in process_key_command()
   - Writes same frame data to pieces/gl_display/current_frame.txt

### 3. Create gl_renderer.c
   - Uses GL/glut.h and GL/glu.h (like 0.gl.emoji.ref❤️]a0.c)
   - Uses FreeType with NotoColorEmoji.ttf
   - Reads pieces/gl_display/current_frame.txt
   - Parses ASCII frame and renders each character as emoji
   - Opens GL window, renders frames at ~60fps
   - Monitors file for changes (like terminal renderer)

### 4. Update compile_all.sh
   - Add gcc command to compile gl_renderer.c → +x/gl_renderer.+x
   - Link against GL, glut, freetype libraries

### 5. Update orchestrator.c (optional)
   - Add gl_renderer thread if running both renderers
   - Or create separate run script for GL mode

### 6. Create run_gl_fuzzpet.sh
   - Script to run the GL version
   - Similar to run_fuzzpet.sh but launches gl_renderer instead of renderer

## File Format for gl_display/current_frame.txt
Same as display/current_frame.txt:
```
==================================
       FUZZPET DASHBOARD        
==================================
Pet Name: Fuzzball
Hunger: 50
...
GAME MAP:
####################
#  ...............T#
...
Controls: w=up, s=down...
==================================
```

## GL Rendering Strategy
- Parse the text file line by line
- For each character, look up the corresponding emoji glyph
- Use the render_emoji() function from 0.gl.emoji.ref❤️]a0.c as reference
- Position characters in a grid layout using glOrtho
- Support basic ASCII characters mapped to emoji equivalents

## Character to Emoji Mapping
- Letters (a-z, A-Z) → Use corresponding Unicode letters or generic emoji
- Numbers (0-9) → Use number emoji 0️⃣1️⃣2️⃣...
- @ (player) → 🧑 or 😊
- # (wall) → 🧱 or ⬛
- . (space) → ⬜ or empty
- T (treasure) → 💎 or ⭐
- R (rock) → 🪨 or 🗿
- Other ASCII → Use Noto Color Emoji equivalents

## TPM Compliance
- gl_display is a proper piece with piece.txt, ledger.txt, history.txt
- All state changes logged to master_ledger
- File-based communication only (no shared memory)
- gl_renderer is stateless - only reads and renders
