The toggle button/key should switch between ASCII and emoji rendering 
modes on the game map, but it's not working.

## REFERENCE BEHAVIOR (from look-emoji+op-ed.txt)

- When ON: Map renders with emojis (🧱🐶🧟🎯 etc)
- When OFF: Map renders with ASCII (#.@&Z etc)
- Response should show: "Emoji Mode ON" or "Emoji Mode: ON/OFF"

## ROOT CAUSE ANALYSIS

### ISSUE 1: fuzz-op missing key handler
- fuzz-op_manager.c doesn't handle key '6' for emoji toggle
- Key '6' gets intercepted by method_key handler (lines 452-454)
- Needs explicit handler like op-ed has for key '4'

### ISSUE 2: render_map.c doesn't support emoji mode
- render_map.c doesn't read emoji_mode from anywhere
- Doesn't convert ASCII glyphs to emojis
- Doesn't check manager state for emoji_mode flag
- Map rendering is completely independent of emoji_mode

### ISSUE 3: No glyph conversion logic
- Need mapping between ASCII and emoji glyphs:
  # <-> 🧱, . <-> ⬛, R <-> 🧟, T <-> 🎯, @ <-> 🐶, etc.
- render_map needs to apply this conversion when emoji_mode=1
