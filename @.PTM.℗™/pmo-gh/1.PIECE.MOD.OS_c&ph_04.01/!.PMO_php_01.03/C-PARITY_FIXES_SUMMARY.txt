## Summary of Changes

### 1. parser.js - Complete Rewrite for C-PARITY

**Key Fixes:**
- **Menu Handling:** Only `<menu>` tag itself is counted as interactive. Children are hidden until menu is opened (activeIndex === menuIndex).
- **Navigation:** Proper focus indicators: `[>]` for focused, `[ ]` for unfocused, `[^]` for active
- **href Navigation:** Full page transition support - when button has href, layout changes and parser resets
- **INTERACT Mode:** All keys forwarded to module's history.txt, ESC (27) exits INTERACT mode
- **Variable Loading:** loadVars() now loads from:
  - `fuzzpet/state.txt` (Mirror runtime state)
  - `clock/state.txt` (Turn and time)
  - `display/state.txt` (Display settings)
  - `fuzzpet/view.txt` (Sovereign view as ${game_map})

**Navigation Behavior (C-PARITY):**
- Arrow keys (1002/1003/1000/1001) or WASD move focus between interactive elements
- Number keys (1-9) jump to element by interactive index
- Enter (13) activates: follows href, opens menu, or triggers onClick
- Nav buffer shows last key pressed ('w'/'s'/'a'/'d')

### 2. orchestrator.js - Map Rendering & Key Processing

**Key Fixes:**
- **Map Rendering:** Exact format matching C version:
  - Each line: `║  <60-char content>║`
  - Map uses: `#` (wall), `.` (floor), `@` (pet), `T` (tree), `R` (rock)
  - Trees at x=17, y<3; Rocks at x=4, 2<y<7
- **Key Processing:** Handles both arrow key codes (1000-1003) and WASD char codes
- **Response Logic:** Event responses match C version:
  - Feed: "Yum yum~ Feeling so happy! *tail wags wildly*"
  - Play: "Wheee~ Best playtime ever!"
  - Sleep: "Zzz... feeling recharged!"
- **State Sync:** All fields persisted to state.txt (Mirror)

### 3. index.html - Integration & Event Handling

**Key Fixes:**
- **Pulse System:** frame_changed.txt size delta detection (C-PARITY heartbeat)
- **Key Mapping:** Browser keys mapped to C-style codes:
  - ArrowUp → 1002, ArrowDown → 1003, ArrowLeft → 1000, ArrowRight → 1001
  - Escape → 27, Enter → 13, Backspace → 127
- **INTERACT Mode:** Keys forwarded to fuzzpet/history.txt as `KEY_PRESSED: <code>`
- **Virtual Commands:** Button onClick mapped to virtual keys:
  - FEED → '2', PLAY → '3', SLEEP → '4', EVOLVE → '5', END_TURN → '6'

### 4. bridge.php - Directory Creation

**Key Fix:**
- write/append operations now create parent directories if missing (mkdir -p equivalent)

### 5. State Files - Initialization

**Created/Reset:**
- `pieces/display/state.txt` - frame_history=on
- `pieces/display/frame_changed.txt` - Initial "X\n" marker
- `pieces/apps/fuzzpet_app/fuzzpet/view_changed.txt` - Initial "X\n" marker
- `pieces/apps/fuzzpet_app/fuzzpet/state.txt` - All fields initialized
- `pieces/apps/fuzzpet_app/fuzzpet/history.txt` - Cleared

## How to Test

1. Start PHP server: `bash php_server++.sh`
2. Open browser to: `http://localhost:8080` (or port specified)
3. Test navigation:
   - Press 's' or ArrowDown to move focus down
   - Press 'w' or ArrowUp to move focus up
   - Press '2' to jump to element #2
   - Press Enter on "Launch FuzzPet" to navigate
4. Test INTERACT mode:
   - Navigate to "Control Map" button
   - Press Enter to activate INTERACT
   - Press WASD/arrow keys to move pet (@)
   - Watch map update in real-time
   - Press ESC to exit INTERACT
5. Test action buttons:
   - Navigate to "Feed Pet"
   - Press Enter
   - Watch hunger decrease and response update
