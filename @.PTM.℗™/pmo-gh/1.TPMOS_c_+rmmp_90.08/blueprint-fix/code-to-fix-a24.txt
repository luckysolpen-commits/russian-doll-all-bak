# Proof of Fault & Unified Fix: Fuzz-Op-GL Movement (A24)

## 1. Theoretical Comparison (The Discrepancy)

| Stage | Fuzz-Op (ASCII/Perfect) | Fuzz-Op-GL (Faulty) | Fault Analysis |
| :--- | :--- | :--- | :--- |
| **Input** | `history.txt` | `session/history.txt` | **Split:** Logic lives in two threads/files. |
| **State** | `xlector/state.txt` | Local `cam_pos[]` floats | **Broken Mirror:** GL window shows manager variables, not piece state. |
| **Movement** | `move_entity.+x` (Op) | `cam_pos[2] += speed` (Hardcoded) | **Thin Brain Violation:** Manager is doing the "muscle" work. |
| **Sync** | Pulse Marker (M) | Pulse Marker (G) | **Desync:** Camera moves but Xelector (piece) stays still. |

## 2. The Fault Proof
In `fuzz-op-gl_manager.c`, the `gltpm_input_thread` hijacks `w,a,s,d` when `is_map_control` is active. It updates local `cam_pos` floats but **fails to call `move_entity.+x xlector`**. 

**Result:** The GL camera flies away while the logical "Xelector" (which GL-OS draws based on the `xlector` piece) remains at its original coordinates. The user sees a "ghost" cursor or nothing at all.

## 3. Definitive Fix Proof
To restore the "Perfect Beautiful" flow:
1.  **Unify Input:** `gltpm_input_thread` must only translate project-history to `route_input()`.
2.  **Unify Movement:** `route_input()` must call `move_entity.+x` for BOTH entity movement and camera movement.
3.  **Mirror Camera:** The camera should be treated as a Piece (or virtual Piece in `session/state.txt`) that `move_entity` or a new `move_camera` op updates.
4.  **Shadow Movement:** When in Map Control, moving the camera SHOULD move the `xlector` piece to keep the "Eye" and the "Finger" aligned.

## 4. Fix Logic (C-Code)
```c
// Inside route_input(int key):
if (is_movement_key(key)) {
    // 1. Move the logical entity (Xlector) - updates world state
    run_op("move_entity.+x", active_target_id, dir);
    
    // 2. If in GL Map Control, also update the session camera
    if (is_map_control) {
        update_session_camera_from_entity(active_target_id);
    }
}
```
