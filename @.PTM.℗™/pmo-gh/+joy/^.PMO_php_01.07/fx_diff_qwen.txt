## THE CORE ISSUE: Menu Index → Game Hotkey Parity

You're absolutely right. The critical issue is **how menu indices map to game hotkeys** in both NAVIGATION and INTERACT modes.

### The C Version Flow:

**Layout (main.chtpm) interactive indices:**
```
1. [Control Map]    onClick="INTERACT"
2. [Feed Pet]       onClick="FEED"
3. [Play]           onClick="PLAY"
4. [Sleep]          onClick="SLEEP"
5. [Evolve]         onClick="EVOLVE"
6. [End Turn]       onClick="END_TURN"
7. [Exit to Menu]   href="..."
```

**NAVIGATION Mode (active_index = -1):**
```
User presses '2' → do_jump(2) → focus_index = element[2] (Feed Pet button)
User presses Enter → send_command("FEED") → inject_raw_key('2') + inject_raw_key(2001)
                     → fuzzpet receives BOTH '2' AND J-B1 → feeds pet
```

**INTERACT Mode (active_index points to Control Map with onClick="INTERACT"):**
```
User presses '2' → inject_raw_key('2') → fuzzpet receives '2' → feeds pet
User presses J-B1 (2001) → inject_raw_key(2001) → fuzzpet receives 2001 → feeds pet
```

**The Key Insight:** The interactive_idx (2, 3, 4, 5, 6) IS the hotkey mapping. 
- Index 2 = '2' = Feed
- Index 3 = '3' = Play
- etc.

## CRITICAL BUG #6: Nav Buffer Cleared Before Render (Timing Bug)

### The Problem

In `index.html` `renderLoop()` (lines 148-151):

```javascript
if (clearNavOnNext) {
    parser.navBuffer = "";      // <-- CLEARED BEFORE COMPOSE!
    clearNavOnNext = false;
}

const currentFrame = await parser.composeFrame(layout.content);
```

This clears the nav buffer **before** `composeFrame()` is called, so the buffer is always empty when rendered!

### C Version Flow (chtpm_parser.c lines 586-596):

```c
if (dirty || clear_nav_on_next) { 
    compose_frame();            // <-- 1. Compose WITH nav_buffer content
    // ...
    if (clear_nav_on_next) { 
        nav_buffer[0] = '\0';   // <-- 2. THEN clear AFTER composing
        clear_nav_on_next = false; 
        compose_frame();        // <-- 3. Compose again to show cleared state
    } 
}
```

The C version shows the key for **one frame** before clearing. This is the "clear_nav_on_next" mechanism - show the last key for one frame, then clear.

### PHP/JS Bug Flow:

```
1. User presses 'w'
2. handleInput() sets: navBuffer = "w", clearNavOnNext = true
3. renderLoop() runs:
   - Clears navBuffer = "" (BEFORE compose!)
   - composeFrame() sees empty buffer
   - Frame shows empty nav buffer
4. User never sees "w" in the nav buffer!
```

### Impact:
- Nav buffer always appears empty
- User cannot see what key they pressed
- Breaks the visual feedback mechanism

### Fix Required:

