# Investigation Report: Corrupt Files and Clock Anomalies
## Date: 2026-03-06

### 1. Root Causes Identifed
Two critical flaws were found in the current (13.01) codebase compared to the Golden (06.00) version.

#### A. Memory Corruption (Use-After-Free)
The corrupted filenames (e.g., `D-]`) appearing in the root are caused by a severe bug in `increment_time.c` and `increment_turn.c`.
- **Finding:** Both utilities call `free(state_path)` *before* calling `fopen(state_path, "w")`.
- **Impact:** `fopen` receives a garbage pointer, creating files with erratic names. The content of these files contains "turn=", "time=", and "mode=", which explains why they "say clock."
- **Solution:** Move `free(state_path)` to the end of the `main()` function.

#### B. Path Mismatch (location_kvp)
The system is writing into "main" (the version root) because the `location_kvp` file points to `^.PMO_c_+e&j8_13.00` instead of the current `13.01` directory.
- **Finding:** `orchestrator.c` performs a `chdir()` based on the `project_root` found in `location_kvp`. Since this path is wrong, all child processes (manager, clock, etc.) are executing in the wrong directory.
- **Golden Proof:** In version 06.00, `orchestrator.c` used simple relative paths (`./`) and `run_chtpm.sh` correctly generated `location_kvp` using `SCRIPT_DIR`.
- **Solution:** Update `run_chtpm.sh` to correctly resolve the absolute path and ensure it overwrites the stale `location_kvp`.

### 2. Proposed Non-Invasive Solutions

#### FIX 1: Correct increment_time.c & increment_turn.c
```c
// Move free() below the final fopen usage
fclose(f_read);
// free(state_path); <-- REMOVE THIS
...
f_write = fopen(state_path, "w");
...
free(state_path); // <-- MOVE TO HERE
```

#### FIX 2: Correct run_chtpm.sh
Ensure `location_kvp` generation uses a clean `SCRIPT_DIR` resolution:
```bash
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
echo "project_root=${SCRIPT_DIR}" > pieces/locations/location_kvp
```

#### FIX 3: Stabilize orchestrator.c
Align with Golden 06.00 logic by ensuring it defaults to the current working directory if `location_kvp` is missing or invalid, preventing the "main" directory writes.
