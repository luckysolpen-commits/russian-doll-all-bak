## 2. KEY DIFFERENCES

2. **run_command()** - fork/exec/waitpid wrapper
3. **Piece selection** - SPACE to select entity at cursor
4. **Context menu** - Mode for piece operations (1-4 keys)
5. **Delete piece** - Call delete_piece.+x op
6. **Undo** - Ctrl+Z calls undo_action.+x
7. **Clear tile** - Backspace clears tile at cursor
8. **Create new map** - ADD functionality in map palette
9. **Focus sync** - Read active_gui_index.txt
10. **Mirror sync** - Update fuzzpet_mirror/state.txt
11. **Full state writing** - Complete state.txt updates

### Different in op-ed (Must Align)
1. **system() calls** → Replace with run_command()
2. **No PDL usage** → Add PDL method dispatch for ops
3. **Simplified main loop** → Add all editor_module features

## 5. CODE PATTERNS TO COPY

### From editor_module.c → op-ed_module.c

#### Signal Handling (Line 26-48)
```c
static volatile sig_atomic_t g_shutdown = 0;

static void handle_sigint(int sig) {
    (void)sig;
    g_shutdown = 1;
}

static int run_command(const char* cmd) {
    pid_t pid = fork();
    if (pid == 0) {
        freopen("/dev/null", "w", stdout);
        freopen("/dev/null", "w", stderr);
        execl("/bin/sh", "/bin/sh", "-c", cmd, (char *)NULL);
        _exit(127);
    } else if (pid > 0) {
        int status;
        waitpid(pid, &status, 0);
        return WIFEXITED(status) ? WEXITSTATUS(status) : -1;
    }
    return -1;
}
```

                    break;
