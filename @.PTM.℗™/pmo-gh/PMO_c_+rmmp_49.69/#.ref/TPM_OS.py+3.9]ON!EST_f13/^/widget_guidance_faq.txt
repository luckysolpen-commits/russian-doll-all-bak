# Widget Integration FAQ for PyOS

## Overview
This document explains how to properly integrate new applications/widgets into the PyOS desktop environment, focusing on the distinction between embedded widgets and separate windowed applications.

## Two Types of Application Launches

PyOS supports two distinct ways to launch applications:

### 1. Context Menu Launch (Right-click on Desktop)
- **Behavior**: Creates a MiniWindow with title bar and borders
- **Example**: Right-click → "Play Hangman" or "2D PyBoard"
- **Implementation Pattern**: Create MiniWindow and add to desktop's `add_window()` method
- **UI**: Shows as a floating window with minimize/maximize/close buttons

### 2. Terminal Command Launch (In-App Terminal)
- **Behavior**: Also creates a MiniWindow that appears in the desktop
- **Example**: Open terminal → type `hangman` or `pyboard2d`
- **Implementation Pattern**: Similar to context menu but initiated from terminal commands
- **Location**: Created by finding parent desktop through widget hierarchy

## Key Implementation Patterns

### For Context Menu (DesktopWidget methods):
```python
def show_new_app(self):
    from apps.new_app import NewApp
    from window_manager.mini_window import MiniWindow
    
    app_widget = NewApp()
    app_window = MiniWindow(title="New App", content_widget=app_widget)
    app_window.setParent(self)
    app_window.show()
    self.add_window(app_window)  # Add to desktop's window tracking
```

### For Terminal Commands (TerminalApp methods):
```python
def run_new_app(self):
    try:
        parent = self.parent()
        # Navigate up widget tree to find desktop
        while parent and not hasattr(parent, 'add_window'):
            parent = parent.parent()
            
        if parent and hasattr(parent, 'add_window'):
            from apps.new_app import NewApp
            from window_manager.mini_window import MiniWindow
            
            app_widget = NewApp()
            app_window = MiniWindow(title="New App", content_widget=app_widget)
            app_window.setParent(parent)
            app_window.show()
            parent.add_window(app_window)  # Add to desktop's window tracking
            self.display_success("Launched New App in desktop")
    except Exception as e:
        self.display_error(f"Failed to launch new_app: {str(e)}")
```

And add command handling in `execute_command()`:
```python
elif cmd == 'newapp':
    self.run_new_app()
    self.display_prompt()
```

### For Adding to Terminal Help:
```python
help_text = "\nAvailable commands:\n  ... \n  newapp   - Launch new app (creates window in desktop)"
```

## Important Methods Added to DesktopWidget

### add_widget_to_desktop(widget)
- **Purpose**: Adds widgets directly to desktop layout (for future use)
- **When to use**: When creating embedded widgets that don't need window frames
- **Note**: Not used for MiniWindow apps but available for other embedding needs

## Common Mistakes to Avoid

1. **Confusing embedding vs. windowed**: All context menu apps should create MiniWindows, not embed directly
2. **Missing parent navigation**: Terminal commands need to traverse the widget hierarchy to find the desktop
3. **Forgetting to call add_window()**: This tracks the window in the desktop's window list
4. **Not matching behavior**: Terminal command launch should create the same kind of window as context menu
5. **Size policy issues**: Child widgets should use QSizePolicy.Expanding to allow proper resizing

## Adding New Apps Checklist

1. Register the app in main.py AppManager
2. Add context menu handler in DesktopWidget (creates MiniWindow)
3. Add terminal command handler in TerminalApp (creates MiniWindow)
4. Add command to execute_command method in TerminalApp
5. Update help text in TerminalApp
6. Test both launch methods ensure consistent behavior

## Making Widgets Resizable

For widgets to properly resize within MiniWindows:

1. Set QSizePolicy.Expanding on main widget: `self.setSizePolicy(QSizePolicy.Expanding, QSizePolicy.Expanding)`
2. For widgets with fixed minimum sizes, allow expansion beyond minimum
3. Ensure parent layouts accommodate dynamic sizing
4. Be cautious with layout additions like addStretch() that might interfere with specific widget layouts
