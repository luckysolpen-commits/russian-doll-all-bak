================================================================================
COMMON PITFALLS (READ THESE FIRST!)
================================================================================

#72: GL-OS IS APP-ONLY
- Don't delete projects/gl-os/ without checking reference
- GL-OS launched via LAUNCH:GL-OS, not LOAD_PROJECT

#73: ${desktop_view} NEEDS SPECIAL HANDLING
- Multi-line values don't work in state.txt
- Parser reads ${desktop_view} directly from view.txt

#74: REFERENCE-FIRST DEBUGGING
- ALWAYS check reference before making changes
- We wasted 3 hours by not doing this

#75: APP vs PROJECT CONFUSION
- APP: System tools (GL-OS, man-ops)
- PROJECT: User content (op-ed, fuzz-op)
- Different patterns for each

#76: MULTI-LINE VALUES IN STATE FILES
- State files split on '=' character
- Use | delimiter or read directly from source

#65: REFERENCE RESTORATION POLICY
- When functionality missing, restore from reference first
- Compare line counts: wc -l current.c #.ref/.../reference.c
- If different: cp #.ref/.../reference.c current/ && recompile && test

#66: GL-OS APP STRUCTURE
- GL-OS requires pieces/apps/gl_os/ structure
- Has layouts/, plugins/, session/, state/, windows/
- OpenGL compilation requires: -lGL -lglut -lGLU

================================================================================
FILE STRUCTURE REFERENCE
================================================================================

APP Structure (pieces/apps/<app_id>/):
  ├── <app_id>.pdl (APP descriptor)
  ├── layouts/ (.chtpm files)
  ├── plugins/ (+x/ for binaries)
  ├── session/ (runtime state)
  └── state/ (persistent state)

PROJECT Structure (projects/<project_id>/):
  ├── project.pdl (entry_layout, starting_map)
  ├── manager/+x/<project>_manager.+x
  ├── maps/ (editable maps)
  ├── pieces/ (entities with state.txt)
  └── layouts/ (.chtpm files)

SHARED Infrastructure:
  ├── pieces/os/app_list.txt (APP routes)
  ├── pieces/os/project_routes.kvp (PROJECT routes)
  ├── pieces/apps/player_app/manager/state.txt (engine state)
  └── pieces/apps/player_app/history.txt (input buffer)

================================================================================
WHEN STUCK (FOLLOW THIS ORDER)
================================================================================

1. CHECK REFERENCE
   ls #.ref/TPMOS_c_+rmmp_52.00/<path>/
   Does reference have this structure?

2. CHECK DOCUMENTATION
   grep -i "keyword" #.docs/^.pmo.ld-faq+8/PITFALLS_ACTIVE_2026-03-18.txt
   Is this a known pitfall?

3. CHECK STATE FILES
   cat pieces/.../state.txt
   Are variables being written correctly?

4. CHECK LAYOUT VARIABLES
   grep "\${" layout.chtpm
   Are all variables defined in state?

5. CHECK BINARY
   file path/to/binary.+x
   Does binary exist and is executable?

6. TEST IN ISOLATION
   ./path/to/op.+x <args>
   Does op work when called directly?

7. ASK FOR HELP
   - What are you trying to do?
   - What did you expect?
   - What actually happened?
   - What have you checked so far?
