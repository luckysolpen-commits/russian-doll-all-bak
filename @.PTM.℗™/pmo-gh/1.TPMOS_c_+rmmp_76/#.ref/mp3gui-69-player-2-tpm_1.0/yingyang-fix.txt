# YingYang Player Fix Plan

## Issues Identified
1. **Enter Key**: The `Enter` key in the GUI only plays a song during search mode. In normal mode, it does nothing. It should play the currently highlighted/selected song.
2. **CLI Robustness**: 
    - `yingyang-cli` doesn't check if `player_command.txt` exists at startup, leading to potential `stat` issues.
    - Error reporting in audio initialization is minimal, making debugging difficult if PulseAudio or FFmpeg fail.
3. **Navigation Feedback**: No visual feedback for "Enter to play" in the UI.

## Proposed Fixes

### A. GUI (yingyang-gui.c)
- Modify `keyboard()` to handle `13` (Enter) in non-search mode. It should send `PLAY <current_song_index>` to the CLI.
- Ensure `current_song_index` is updated correctly when navigating with arrows.

### B. CLI (yingyang-cli.c)
- Add a check at startup to create `player_command.txt` if it doesn't exist (mirroring the reference code).
- Add more `printf` logs to `init_audio()` and `audio_thread()` to trace why audio might not be playing.
- Check return values of `pa_simple_new` and `pa_simple_write` and log errors.

### C. Workflow
- Document keys in the UI text.
- Re-compile and test.
