#### ISSUE 1: Stale loader.pdl (Primary Cause)
File: pieces/apps/playrm/loader/loader.pdl
Last Modified: Apr 13 15:21 (BEFORE cyoa-engine was created on Apr 15 07:09)

The loader_module.c daemon scans projects/ at startup and regenerates loader.pdl dynamically.

cyoa-engine/project.pdl EXISTS and is valid, but the loader_module daemon was started

re-scans. It must be RESTARTED to pick up new projects.

Current loader.pdl contains 13 projects (404-test through user) — no cyoa-engine.

#### ISSUE 2: Missing from project_routes.kvp (Secondary Cause)
File: pieces/os/project_routes.kvp

This file maps project names to layout paths for the LOAD_PROJECT command path.
cyoa-engine is NOT listed. If loaded via the parser's LOAD_PROJECT command (rather than

                 ai-labs, p2p-net, lsr, blank

Missing: cyoa-engine, gitlet, 404-test, test_fondu

### HOW EXISTING PROJECTS WORK (Code Standards)

The system uses a PDL-based dynamic discovery pattern — NO hardcoded project lists.

**Pattern 1: loader_module.c (Daemon-based, PRIMARY)**
- Scans projects/ directory at startup
- Filters: skips dotfiles, "trunk" archive, and dirs WITHOUT project.pdl
- Regenerates loader.pdl with METHOD entries for each valid project
- Reads entry_layout from project.pdl to switch layouts on load
- Uses MODULAR FALLBACKS for legacy projects without entry_layout (lines 149-160):
    - op-ed, template → projects/op-ed/layouts/op-ed.chtpm
    - man-ops → pieces/apps/man-ops/layouts/man-ops.chtpm
    - man-pal → pieces/apps/man-pal/layouts/man-pal.chtpm
    - everything else → pieces/apps/playrm/layouts/game.chtpm

**Pattern 2: project_loader.c (On-demand Op, SECONDARY)**
- Two versions exist (playrm/ops/src/ and player_app/world/plugins/)
- Scans projects/ at runtime (no restart needed)
- Does NOT check for project.pdl (less strict than loader_module)
- Generates tmp_projects.pdl, calls menu_op, writes to state.txt
- player_app version reads entry_layout and writes to layout_changed.txt

**Pattern 3: project_routes.kvp (Fallback routing)**
- Used by LOAD_PROJECT command in parser
- If project not listed → shows 404.chtpm
- Should be kept in sync with actual projects

The user explicitly stated preference to NOT hardcode dynamic elements.

### WHY op-ed AND fuzz-op WORK

Both were created BEFORE the loader_module daemon was last started (Apr 13).
They are in the stale loader.pdl file. They also have project.pdl files with
proper structure. op-ed has entry_layout, fuzz-op does NOT (uses fallback).

cyoa-engine follows the SAME pattern as op-ed (has entry_layout in PDL).
It just wasn't there when the daemon last started.
