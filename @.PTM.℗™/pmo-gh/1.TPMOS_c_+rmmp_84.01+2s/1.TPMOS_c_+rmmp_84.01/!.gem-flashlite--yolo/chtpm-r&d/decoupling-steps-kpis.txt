# Decoupling Legacy Dependencies: Steps & KPIs

This document outlines the surgical plan to remove hardcoded legacy dependencies (primarily `fuzzpet_app`) from the TPMOS core, ensuring a clean, project-first architecture.

## 1. PRE-REQUISITES (The Safety Net)
*   **Mandatory Backups**: Before any change, create `.bak` snapshots:
    *   `pieces/chtpm/plugins/chtpm_parser.c` -> `chtpm_parser.c.bak.A22-decouple`
    *   `pieces/buttons/shared/run_orchestrator.sh` -> `run_orchestrator.sh.bak.A22-decouple`

---

## 2. IMPLEMENTATION STEPS

### Phase 1: Registry Sanitization (`run_orchestrator.sh`)
1.  **Remove Legacy Keys**: Delete the following lines from the `location_kvp` generation block:
    *   `fuzzpet_app_dir`
    *   `fuzzpet_dir`
    *   `manager_dir` (which points to the legacy pet manager)
2.  **Add Modern Anchors**: Ensure `chat-op` and `fuzz-op` have entries if they require project-specific system pivots (though the goal is to use `project_root` relative paths).

### Phase 2: Parser Refactoring (`chtpm_parser.c`)
1.  **Neutralize `load_vars()`**:
    *   Remove the unconditional `load_state_file("pieces/apps/fuzzpet_app/fuzzpet/state.txt", "fuzzpet")`.
    *   Change the default fallback in the `is_modern_layout` check. Instead of loading `fuzzpet_app/manager/state.txt`, it should load a system-default state or do nothing.
2.  **Scrub Hardcoded Strings**: Search for all occurrences of "fuzzpet" and ensure they are either moved to the `fuzz-op` special case block or converted to dynamic variables.

### Phase 3: Migration & Validation
1.  **Soft-Delete Test**: Rename `pieces/apps/fuzzpet_app/` to `pieces/apps/fuzzpet_app.OBSOLETE/`.
2.  **Boot Test**: Run `./button.sh compile` and then `./run_chtpm.sh`.
3.  **Project Integrity**: Load `fuzz-op` from the loader and verify stats (hunger, energy, etc.) are correctly read from `projects/fuzz-op/pieces/fuzzpet/state.txt`.

---

## 3. KPIs (Key Performance Indicators)

| KPI | Target | Current Status |
| :--- | :--- | :--- |
| **Registry Purity** | Zero "fuzzpet" keys in `location_kvp`. | 3 Keys Found |
| **Parser Neutrality** | Zero "fuzzpet" hardcoded paths in `chtpm_parser.c`. | ~5 Hardcoded paths |
| **Filesystem Cleanliness** | `pieces/apps/fuzzpet_app` directory deleted. | Still present/required |
| **Boot Success** | System launches to Loader without legacy error logs. | Passes (but relies on legacy) |

---

## 4. TROUBLESHOOTING / ROLLBACK
*   **If UI elements disappear**: Check if `${pet_...}` variables were relying on the legacy `fuzzpet` prefix instead of the new `pet` prefix used in `fuzz-op`.
*   **If Manager fails to launch**: Check if `location_kvp` was the only place the manager-path was defined.
*   **Rollback**: Copy `.bak` files back to their original names and restore the `fuzzpet_app` directory.
