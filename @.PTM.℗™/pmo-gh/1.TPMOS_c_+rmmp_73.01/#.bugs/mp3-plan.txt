    1 # MP3 Store Project Plan - Phase 1 & Future Features
    2 # Document ID: mp3-plan-a20.txt
    3 # Date: April 20, 2026
    4 # Status: PLANNING DOCUMENT
    5
    6 ================================================================================
    7 ## 1. PROJECT GOAL
    8 ================================================================================
    9
   10 To develop a user-friendly MP3 player application within the TPMOS ecosystem.
   11 The initial focus will be on a robust ASCII-based player with directory navigation,
   12 followed by the integration of advanced features like search, visualizers, and
   13 eventually a GL renderer. Monetization through advertisements will be explored.
   14
   15 ================================================================================
   16 ## 2. CORE FUNCTIONALITY (Phase 1 - ASCII Renderer)
   17 ================================================================================
   18
   19 This phase focuses on establishing the foundational player and UI elements using
   20 an ASCII-based CHTPM layout.
   21
   22 ### 2.1 MP3 Player Core
   23 - **Playback Engine:** Basic playback of MP3 files.
   24 - **File Handling:** Ability to read and process MP3 files.
   25
   26 ### 2.2 ASCII CHTPM UI
   27 - **Directory Navigation:**
   28     - Browse file system from the root directory (`projects/mp3-store/music/` as
      initial).
   29     - Display directories and files.
   30     - Open/Collapse directories to reveal/hide contents.
   31     - Navigation using arrow keys.
   32 - **File Selection:**
   33     - Select files for playback.
   34     - ENTER key to play selected MP3 file.
   35 - **Basic Player Controls:**
   36     - Display current playback status (playing, paused, stopped).
   37     - Basic navigation: Play/Pause, Next, Previous (will be mapped to keys later).
   38 - **State Management:**
   39     - Track current directory.
   40     - Track currently playing file.
   41     - Store last known UI state.
   42
   43 ### 2.3 Data Storage & Retention
   44 - **User Music Library:** Allow users to manage their music files within the app's
      designated storage area (`projects/mp3-store/music/`).
   45 - **Playback State:** Persist current playback position and playlist state (future).
   46
   47 ================================================================================
   48 ## 3. FUTURE FEATURES (Post-MVP)
   49 ================================================================================
   50
   51 ### 3.1 Enhanced Playback & UI
   52 - **Playlist Management:** Create, save, load, and manipulate playlists.
   53 - **Directory Collapsing/Expanding:** More sophisticated UI for managing nested
      directories.
   54 - **Visualizer:** Implement an ASCII-based visualizer in the map area.
   55 - **GL Renderer:** Integrate a graphical renderer for a more advanced user interface
      (leveraging GL-OS).
   56
   57 ### 3.2 Search Functionality
   58 - **File Search:** Allow users to search for MP3 files by name, artist, or album.
   59
   60 ### 3.3 Monetization
   61 - **Advertisement Integration:** Explore methods for integrating ads (e.g., banner
      ads in ASCII UI, interstitial ads).
   62
   63 ### 3.4 Extended File Support
   64 - **Other Audio Formats:** Support for formats beyond MP3 (e.g., OGG, FLAC).
   65
   66 ================================================================================
   67 ## 4. ARCHITECTURAL CONSIDERATIONS
   68 ================================================================================
   69
   70 - **CPU Safety:** Adhere to TPM CPU-safe patterns (signal handling,
      `fork/exec/waitpid`, focus throttling).
   71 - **File-Based State:** All state (playback position, playlists, settings) will be
      file-backed for auditability and persistence.
   72 - **Modularity:** Design components (playback, UI, file browser) to be swappable.
   73 - **Modularity over Hardcoding:** Avoid hardcoded paths or behaviors where dynamic
      resolution is possible (e.g., using `location_kvp` for paths).
   74
   75 ================================================================================
   76 ## 5. DEVELOPMENT PRIORITIES
   77 ================================================================================
   78
   79 1.  **Core ASCII Player & Directory Navigation:** Establish basic functionality
      first.
   80 2.  **Playlist Management:** Essential for a music player.
   81 3.  **Search Functionality:** Improves usability significantly.
   82 4.  **GL Renderer:** Long-term goal for visual enhancement.
   83 5.  **Monetization:** Explore ad integration once core features are stable.
