## Chat App Project Plan (Detailed)

**Objective:** To create a functional chat application within the TPMOS ecosystem, leveraging existing p2p operations for real-time communication.

**Key Performance Indicators (KPIs):**
*   **Message Delivery:** Messages sent by one user are reliably received by other connected users within 5 seconds (under normal network conditions).
*   **UI Responsiveness:** The chat interface remains responsive, with message display updating within 1 second of reception.
*   **User Identification:** Users can be uniquely identified (e.g., by a username).
*   **Project Integration:** The chat app project successfully compiles, installs via Fondu, and loads via the TPMOS project loader.

---

### Phase 1: Core Infrastructure Setup

**Goal:** Establish the foundational project structure, descriptor, and basic manager/op integration.

*   **Step 1.1: Finalize `project.pdl`**
    *   **Action:** Review and confirm the `project.pdl` file content for accuracy and completeness. (Already populated)
    *   **Deliverable:** Verified `projects/chat-app/project.pdl`.
*   **Step 1.2: Develop `chat-app_manager.c` (Initial Structure)**
    *   **Action:** Create the basic C file for the manager. Implement essential TPMOS patterns: signal handling, basic input polling, and placeholder functions for op interaction and rendering triggers.
    *   **Deliverable:** `projects/chat-app/manager/chat-app_manager.c` with initial structure.
*   **Step 1.3: Integrate P2P Operations**
    *   **Action:** Identify specific p2p ops relevant for chat (e.g., network connectivity, message broadcast). Register these ops in `projects/chat-app/ops/ops_manifest.txt`. If stubbed ops are required, create minimal C files in `projects/chat-app/ops/src/`.
    *   **Deliverable:** `projects/chat-app/ops/ops_manifest.txt` populated with op registrations. Minimal op source files (if needed).
*   **Step 1.4: Initial Compilation and Registration**
    *   **Action:** Compile the project using `./button.sh compile`. Install the project using `pieces/system/fondu/fondu.+x --install chat-app`. Verify successful registration and no initial errors.
    *   **Deliverable:** Project compiled and installed. Verified in TPMOS loader.

---

### Phase 2: Basic Chat Functionality

**Goal:** Enable sending and receiving of simple text messages between users.

*   **Step 2.1: Implement Message Sending**
    *   **Action:** Modify `chat-app_manager.c` to capture user input (e.g., from a basic text input in the layout) and pass it to a designated P2P op for broadcasting.
    *   **Deliverable:** Manager can send text messages via an op.
*   **Step 2.2: Implement Message Receiving**
    *   **Action:** Modify the manager to receive broadcasted messages from the P2P op and process them.
    *   **Deliverable:** Manager can receive text messages from other users.
*   **Step 2.3: Design Chat UI Layout (`chat-app.chtpm`)**
    *   **Action:** Create `projects/chat-app/layouts/chat-app.chtpm`. Define the structure for displaying messages (e.g., a scrollable message pane) and an input field.
    *   **Deliverable:** Basic `.chtpm` layout file for the chat interface.
*   **Step 2.4: Display Messages in UI**
    *   **Action:** Update the manager to populate the chat layout (`.chtpm`) with received messages. Use TPMOS variable substitution (e.g., `${message_list}`) if applicable.
    *   **Deliverable:** Messages are displayed in the chat UI.
*   **Step 2.5: Testing Phase 2**
    *   **Action:** Test sending and receiving messages between at least two instances (simulated or actual). Verify message delivery and display.
    *   **Deliverable:** Basic end-to-end message functionality confirmed.

---

### Phase 3: UI Refinements and User Management

**Goal:** Enhance the user experience with user identification and improved message display.

*   **Step 3.1: Implement User Identification**
    *   **Action:** Allow users to set a username during project startup or via a configuration. Ensure messages are tagged with the sender's username.
    *   **Deliverable:** Usernames are associated with sent messages.
*   **Step 3.2: Enhance Message Display**
    *   **Action:** Refine the `chat-app.chtpm` layout to display messages with sender names and timestamps.
    *   **Deliverable:** Improved message rendering in the UI.
*   **Step 3.3: Basic User Presence/List (Optional)**
    *   **Action:** If time/scope permits, implement a simple mechanism to show connected users.
    *   **Deliverable:** Basic user list or presence indicator.

---

### Phase 4: Testing, Compilation, and Deployment

**Goal:** Ensure the chat application is robust, well-tested, and ready for use.

*   **Step 4.1: Unit and Integration Testing**
    *   **Action:** Develop tests for core manager functions and op interactions.
    *   **Deliverable:** Test suite for the chat application.
*   **Step 4.2: Manual Testing and Bug Fixing**
    *   **Action:** Conduct thorough manual testing of all features. Address any bugs identified.
    *   **Deliverable:** Stable and functional chat application.
*   **Step 4.3: Final Compilation and Installation**
    *   **Action:** Re-run `./button.sh compile` and `pieces/system/fondu/fondu.+x --install chat-app` to ensure everything is correctly built and registered.
    *   **Deliverable:** Fully compiled and installed chat application.
