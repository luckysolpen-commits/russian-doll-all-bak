## 1. CORE NUANCES TO MIRROR

The Legacy FuzzPet (`fuzzpet_app`) had several small but vital features that gave it "life." We must port these to the **Shared Ops** library so the Editor can instantiate and test them.

### 1.1 Action-Driven Stat Decay
- **Legacy:** Every move cost 5 Energy and increased Hunger by 2.
- **Ops Plan:** Modify `move_entity.c` or create a wrapper script that calls `move_entity` followed by `modify_stat.+x`.
- **Editor Tool:** Add a "Stat Cost" field in the Piece Props menu.

### 1.2 The "Irregular Verb" Response System
- **Legacy:** `feed` -> "fed", `sleep` -> "slept".
- **Ops Plan:** Standardize `fuzzpet_action.c` to look up responses in the Piece's `.pdl` but handle the tense mapping internally or via a generic `get_response` Op.
- **Editor Tool:** A visual editor for the `RESPONSE` section of a Piece's PDL.

### 1.3 Emoji/Glyph Duality
- **Legacy:** Toggle between ASCII and Emojis.
- **Ops Plan:** `render_map.c` already supports `emoji_mode`. We need to ensure the Editor's `emoji_mode` variable is synced to the project's `state.txt` so the Player respects it.
