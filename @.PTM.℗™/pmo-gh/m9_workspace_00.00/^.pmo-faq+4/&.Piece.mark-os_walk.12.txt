-----------------------------------
Every journey in Piecemark begins with `@^.PMO_c_+e&j8_4.00/run_chtpm.sh`.
- It acts as the "BIOS" of the system.

2. HOW TO READ THE CODEBASE (The Data Path)
-------------------------------------------
To understand how a keypress becomes a pixel, follow this 12-step pipeline:

1. INPUT: `pieces/keyboard/plugins/keyboard_input.c` writes ASCII to `history.txt`.
2. ROUTING: `pieces/chtpm/plugins/chtpm_parser.c` (The Theater) reads the key.
3. RELAY: If in INTERACT mode, the Parser "injects" the key into the App's local `history.txt`.
4. TICK: The App Manager (`pieces/apps/fuzzpet_app/manager/fuzzpet_manager.c`) polls its history.
5. TRAITS: The Manager calls a "Trait" plugin (e.g., `move_entity.c`) to process logic.
6. SOVEREIGNTY: The Trait updates the specific Entity Piece (e.g., `pet_01.pdl`).
7. SYNC: The `piece_manager` immediately flushes the update to `pet_01/state.txt` (The Mirror).
8. STAGE: The Stage Producer (`map_render.c`) reads all Mirrors and writes `view.txt`.
9. **BACKGROUND (v0.3):** `clock_daemon.+x` ticks time (auto mode) or waits for action (turn mode).
10. COMPOSITION: The Parser substitutes `${game_map}` with the contents of `view.txt` AND loads `${clock_time}`, `${clock_turn}`.
11. RENDER: The Parser writes the final frame to `current_frame.txt`.
12. SIGNAL: Daemon/Manager hits `frame_changed.txt` to trigger the renderer.

4. KEY ARCHITECTURAL SYMBOLS
----------------------------
- PIECE: A directory containing a soul (.pdl) and a mirror (state.txt).
- MODULE: A logic agent that animates pieces (e.g., fuzzpet_app).
- OS/THEATER: The CHTPM parser and layout engine.
- **DAEMON (v0.3):** Background process that ticks continuously (e.g., `clock_daemon.+x`).
- MASTER LEDGER: The centralized audit trail of every event in reality.

5. NEW (v0.3): SYSTEM DAEMONS
------------------------------
System daemons are background processes that run independently:
- **clock_daemon:** Manages time/turn based on mode (auto vs turn)
- **Auto mode:** Time ticks every second, turn every 60 seconds
- **Turn mode:** Waits for user action, then advances both time and turn
- **Critical:** Must hit `frame_changed.txt` to trigger UI updates
