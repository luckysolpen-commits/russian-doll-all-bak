# 2D View Implementation Plan - AOW-2D (8x8 Tiles)
Date: 2026-04-28
Status: PLANNED (Execution Phase)

## 🎯 Objective
Create a "purely 2D" tile-based view for `aow-2d` that renders ASCII map characters as 8x8 blocks in the terminal. This uses high-resolution texture data (8x8 RGB CSVs) converted to ANSI ASCII art.

## 🛠️ Strategy

### 1. Fix Project Infrastructure
*   **Target Directory:** `projects/aow-2d/pieces/world/map_01/`
*   **Action:** Modify `projects/aow-2d/ops/gen_map.c` to use relative paths and run it to ensure `map_01_z0.txt` exists.

### 2. Implement `render_2d_map.c`
*   **Input:** `map_01_z0.txt` (ASCII map).
*   **Asset Mapping:**
    *   `.` -> `#.ref/^.fin_mc_8&16=99_c2+]FIXD/mc_extracted_csvs_8x8/water/water.csv`
    *   `#` -> `#.ref/^.fin_mc_8&16=99_c2+]FIXD/mc_extracted_csvs_8x8/grass/grass.csv`
    *   `A` -> `#.ref/^.fin_mc_8&16=99_c2+]FIXD/mc_extracted_csvs_8x8/bricks/bricks.csv` (or similar)
*   **Conversion Logic (Pixel to ASCII):**
    *   Read 8x8 RGB values.
    *   For each "pixel" in the CSV, map it to an ANSI background color.
    *   Example: `\033[48;2;R;G;Bm  \033[0m` (using two spaces for roughly square "pixels").
*   **Composition:**
    *   Loop through the 25x25 map.
    *   Build a large buffer representing the entire map at high detail.
    *   Note: A 25x25 map at 8x8 per tile is 200x200 terminal characters. This may be too large for some terminal views, so we might scale down to 4x4 if necessary, but we'll try 8x8 first.

### 3. Integrate with `aow-2d_manager.c`
*   **Trigger:** Manager calls `render_2d_map.+x` on pulse.
*   **State:** Export the rendered high-detail string to `manager/state.txt` as `map_view`.
*   **UI:** `main.chtpm` displays `${map_view}`.

### 4. GL-OS Parity
*   Update `gl_desktop.c` to recognize `aow-2d` and force a top-down orthographic camera mode when the window is active.

## 🏁 Verification Steps
1.  Verify `map_01_z0.txt` generation.
2.  Compile `render_2d_map.c`.
3.  Check `state.txt` for the `map_view` string.
4.  Launch and visually confirm the "Mini-Map" style 2D rendering.
