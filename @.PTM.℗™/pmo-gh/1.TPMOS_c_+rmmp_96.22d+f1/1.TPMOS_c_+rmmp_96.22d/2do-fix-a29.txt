# TPMOS Fix Plan - A29 Emergent Issues
Date: 2026-04-29
Status: PLANNED

## 🎯 Objectives
1.  **Loader Cleanup:** Remove `aow-2d` from the ASCII Project Loader (`pieces/apps/playrm/loader/loader.pdl`) as it is a GL project.
2.  **GL-OS Decoupling:** Move hardcoded GL projects in `pieces/apps/gl_os/plugins/gl_desktop.c` to a KVP configuration file (`pieces/apps/gl_os/gl_os_projects.kvp`).
3.  **2D Rendering Enforcement:** Force a top-down camera (mini-map style) for `aow-2d` when viewed in GL-OS.
4.  **Bible Update:** Add "Project Type Pollution" pitfall to `!.TPMOS_ONBORD_BIBLE_2.md`.

## 🛠️ Strategy

### 1. Loader Cleanup
*   Edit `pieces/apps/playrm/loader/loader.pdl` to remove the `aow-2d` method entry.

### 2. GL-OS KVP Implementation
*   Create `pieces/apps/gl_os/gl_os_projects.kvp` with mappings for `fuzz-op-gl`, `op-ed-gl`, `aow-2d`, and `piececraft-3d`.
*   Modify `gl_desktop.c`:
    *   Add a function `load_gl_projects()` to read this KVP.
    *   Dynamically populate `menu_options` in `CTX_DESKTOP_APPS` mode.
    *   Update `execute_option` to use the loaded project list.

### 3. Force 2D View for aow-2d
*   In `gl_desktop.c`, within the `project_mirror` or `gltpm_app` rendering loop:
    *   Check if `win->project_id` is `aow-2d`.
    *   If so, override `cam_rot` to `(-90, 0, 0)` (purely top-down).
    *   Optionally disable Z-extrusion or use flat tiles for this project.

### 4. Bible Update
*   Add Pitfall #6: **Project Type Pollution**. "Never list a GL-only project in the ASCII Project Loader. Keep theaters separate to prevent 404/rendering errors."
