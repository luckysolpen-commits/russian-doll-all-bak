GL-OS INTERACTIVE DESKTOP PLAN (M21)
=======================================

1. ARCHITECTURAL OVERHAUL: PROJECT-BASED OPS
-------------------------------------------
Instead of a simple app, GL-OS will follow the `projects/` pattern used by `op-ed`.
- Path: `projects/gl-os/`
- Structure:
    - `project.pdl`: Defines desktop global settings (resolution, wallpaper).
    - `manager/`: Contains `gl_os_manager.+x` (the Brain).
    - `pieces/`: Each window is technically a "Piece" tracked in the project.
    - `history.txt`: Unified input buffer for mouse/keyboard events.

2. FEATURE IMPLEMENTATION
-------------------------

A. MOVEABLE WINDOWS (THE DRAG PIPELINE)
- Renderer (`gl_os_renderer.c`):
    - Implement `glutMouseFunc` and `glutMotionFunc`.
    - Detect clicks. If `y` is within the top 20px of a window, mark as "potential drag".
    - Write `DRAG_START: win_id, x, y` to `history.txt`.
    - Write `DRAG_MOVE: x, y` while mouse is down.
- Manager (`gl_os.c`):
    - Poll `history.txt`.
    - On `DRAG_MOVE`, update window `x,y` in `state.txt` and `view.txt`.
    - The renderer auto-updates because it watches `view_changed.txt`.

B. CONTEXT MENU (RIGHT-CLICK)
- Renderer: Capture `GLUT_RIGHT_BUTTON`. Write `CONTEXT_MENU: x, y` to `history.txt`.
- Manager: 
    - Upon receiving `CONTEXT_MENU`, create a special "Piece" (window) at `x,y` with a `menu` type.
    - Content: "1. Open Terminal | 2. New Project | 3. Kill App".
    - These options are buttons in the virtual window.

C. TERMINAL WINDOWS
- A "Terminal" window in GL-OS is a window whose `content_source` is set to `pieces/display/current_frame.txt`.
- Since `chtpm_parser` constantly updates `current_frame.txt`, the GL window will show a live-updating CLI.

D. THE "OPS" VERSION (PROJECT LOADER)
- To implement this like `op-ed`:
    - Create `projects/gl-os/project.pdl`.
    - Use `pieces/apps/playrm/ops/src/move_entity.c` logic to move window "pieces".
    - This allows us to use the same "Muscle" code for moving characters in FuzzPet and moving windows on the Desktop.
