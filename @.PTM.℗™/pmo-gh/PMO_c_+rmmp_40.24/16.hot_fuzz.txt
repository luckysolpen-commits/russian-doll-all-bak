## THE GOAL

Replicate fuzzpet_app's nav menu method button behavior in fuzz-op:

1. Nav menu shows method buttons: `[scan][collect][place][inventory][toggle_emoji][toggle_clock]`
2. Navigate to button with LEFT/RIGHT arrow keys
3. Press ENTER to execute the method
4. Response shows (e.g., "Found: pet_02 zombie_01 pet_01 selector")

---

## HOW IT WORKS (fuzzpet_app reference)

### Layout File (`fuzzpet_v2.chtpm` or similar):
```xml
<text label="║  ${piece_methods} ║" /><br/>
```

### PDL File (selector.pdl or piece-specific):
```pdl
METHOD       | scan               | pieces/apps/playrm/ops/+x/scan_op.+x selector
METHOD       | collect            | pieces/apps/playrm/ops/+x/collect_op.+x selector
METHOD       | place              | pieces/apps/playrm/ops/+x/place_op.+x selector
METHOD       | inventory          | pieces/apps/playrm/ops/+x/inventory_op.+x selector
METHOD       | toggle_emoji       | echo "toggled"
METHOD       | toggle_clock       | echo "clock toggled"
```

### chtpm_parser.c Behavior:
1. Reads active target's PDL file
2. For each METHOD entry, creates a button with onClick handler
3. `${piece_methods}` is replaced with button HTML
4. When you navigate to button and press ENTER, onClick is executed

### Key Insight:
**The nav buttons work for selector because selector has a PDL file with METHOD entries!**

## ROOT CAUSE

### Issue 1: Selector PDL Missing METHOD Entries
**File:** `projects/fuzz-op/pieces/selector/piece.pdl`

The selector needs METHOD entries for the nav buttons to work:
```pdl

METHOD       | collect            | pieces/apps/playrm/ops/+x/collect_op.+x selector
METHOD       | place              | pieces/apps/playrm/ops/+x/place_op.+x selector
METHOD       | inventory          | pieces/apps/playrm/ops/+x/inventory_op.+x selector
METHOD       | toggle_emoji       | echo "toggled"
METHOD       | toggle_clock       | echo "clock toggled"
```

### Issue 2: Hotkey Code Blocks Selector
**File:** `projects/fuzz-op/manager/fuzz-op_manager.c` line 373

```c
if (method_key > 0 && strcmp(active_target_id, "selector") != 0) {
```

This prevents hotkeys 2-8 from working when selector is active. But the reference shows scan working with selector!
