## 🆚 KEY DIFFERENCES

| Aspect | fuzzpet_app | fuzz-op (enhanced) |
|--------|-------------|------------------------|
| **Storage** | World-based (`pieces/world/map_01/`) | Project-based (`projects/{id}/pieces/`) |
| **State Updates** | piece_manager.+x only | Hybrid (piece_manager + direct) |
| **PDL** | Full implementation | Reader implemented, writers pending |
| **Hotkeys** | Full (2-8 with dynamic UI) | Partial (2-8, static UI) |
| **Z-levels** | 0-1 (2 levels) | 0-2 (3 levels: surface, underground, sky) |
| **Map Filtering** | None (all pieces in one world) | map_id based (pieces tied to specific maps) |
| **Frame Rendering** | Every input | Deduplicated (only on change) |

## 📝 WHAT WAS ADDED (That fuzzpet_app Doesn't Have)

1. **map_id System**
   - Pieces can be tied to specific maps
   - Prevents "bleed-through" across maps
   - Selector inherits entity's map_id on select

2. **3-Level Z System**
   - fuzzpet_app: Z=0-1 (2 levels)
   - fuzz-op: Z=0-2 (3 levels: surface, underground, sky)

3. **Frame Deduplication**
   - render_map.c compares new view with old
   - Only writes if changed
   - Reduces unnecessary renders

4. **PDL Reader Utility**
   - Standalone binary for PDL operations
   - Can be used by any module
   - More flexible than piece_manager for reading
