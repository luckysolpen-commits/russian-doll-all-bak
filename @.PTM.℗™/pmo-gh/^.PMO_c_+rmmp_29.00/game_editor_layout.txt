File: game_editor_layout.txt

1. THE CORE BUILDING BLOCKS:
---------------------------
The editor UI is divided into 5 vertical zones. The Map is the "Theater," and everything below it is the "Console."

ZONE A: HEADER (Fixed)
- Project Name, Active Layout ID, and global status.

ZONE B: THE THEATER (Interactive)
- [^] 1. [Control Map]  <-- The primary Nav entry point.
- The 20x10 ASCII Grid  <-- Wrapped in onClick="INTERACT".
- This zone is ALWAYS visible regardless of tab.

ZONE C: THE STATUS BAR (Dynamic)
- [RESP]: Feedback from the last action.
- [Z-LEVEL], [ACTIVE_TOOL], [LAST_KEY].

ZONE D: THE PALETTES (Tab-Dependent)
- This is where "Map Palette" and "Glyph Palette" live.
- They are scrolling arrays: [ < ] Map_001  Map_002 [ Map_003 ] Map_004 [ > ]
- Selection logic: 
    - Enter on a palette entry "arms" it.
    - Arrows/Scroll keys cycle the visible window.

ZONE E: THE COMMAND MENU (Navigation)
- [MAP] [PIECES] [DB] [PROJECT] [SAVE] [PLAY] [EXIT]

2. NUANCE: "CONTROL MAP" VS "PALETTE SCROLL"
-------------------------------------------
A. CONTROL MAP (Action Mode):
- Focus: Moving the selector '@' on the grid.
- Entry: Selecting Nav item 1 or clicking the map.
- Keys: WASD/SPACE.
- Goal: Painting terrain or placing entities.

B. PALETTE SCROLL (Selection Mode):
- Focus: Changing WHICH map or glyph is currently active.
- Entry: Navigating to the Palette section in the Console.
- Keys: Left/Right arrows or specialized scroll keys.
- Goal: Loading a different level or picking a 'Z' (Zombie) instead of a '#' (Wall).
