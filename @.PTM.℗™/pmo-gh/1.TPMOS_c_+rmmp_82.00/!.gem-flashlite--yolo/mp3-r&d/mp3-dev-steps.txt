# MP3-Store Development Journey & Engineering Standards
**Date:** 2026-04-21
**Project:** mp3-store (Restoration)
**Pattern:** Sovereign PDL / Data-Driven Interaction

---

## 1. THE DEVELOPMENT PROCESS

### Phase A: The Audit (Identifying "Legacy Rot")
The original `mp3-store` was "broken by design." It used custom variables (`${mp3_list}`), hardcoded project paths, and had no clean way to handle the Enter key.
- **Decision:** Do not try to fix the custom variable. Burn it down and migrate to the canonical `${piece_methods}` pattern used by `fuzz-op` and `cyoa-engine`.

### Phase B: Infrastructure Fixes
1.  **OP Upgrade:** Updated `play_mp3.c` to support `--pause` and `--resume` using signals (`SIGSTOP`/`SIGCONT`).
2.  **Parser Whitelisting:** Modified `chtpm_parser.c` to recognize the `MP3:` prefix. Previously, the parser would ignore any command it didn't handle internally. By whitelisting the prefix, we allowed "Custom String Injection" via the Enter key.

### Phase C: Manager Logic (The "Brain")
1.  **Sovereign DNA:** The manager now scans `#.mp3-library/` and *writes* its own `piece.pdl` every time the library changes or the state updates.
2.  **Input Normalization:** Fixed the most common pitfall—the "ASCII vs Integer" trap. The manager now handles raw numeric strings (e.g., "50") from the history file, converting them back to logical indices.
3.  **Context Hygiene:** Ensure the manager writes `active_target_id=mp3_player` to both its local state AND the shared `player_app` state to ensure the parser always knows where to look for methods.

## 2. CRITICAL PITFALLS & SOLUTIONS

### Pitfall #1: The Hyphenation Trap
- **Issue:** Project is `mp3-store`, but source was `mp3_store_manager.c`.
- **Symptom:** `compile_all.sh` silently skipped compilation because its regex expects `projects/<name>/manager/<name>_manager.c`.
- **Standard:** **ALWAYS** match the file name to the directory name exactly.

### Pitfall #2: Stale History Poisoning
- **Issue:** Launching the app causes it to immediately play the last song selected in a previous session.
- **Symptom:** `last_pos` in the input thread starts at 0.
- **Standard:** Initialize `last_pos` using `stat()` to the **END** of the history file on startup. Only process *new* keys.

### Pitfall #3: Enter Key "Dead Air"
- **Issue:** Clicking a button with the mouse works, but pressing Enter does nothing.
- **Symptom:** The parser's `send_command` doesn't know how to "send" a custom string like `MP3:PLAY:song.mp3`.
- **Standard:** Use the `inject_command()` helper in the parser. It formats the command as `COMMAND: <cmd>` and drops it in the history file for the manager to see.

### Pitfall #4: Relative Path Drift
- **Issue:** Manager can't find `mpg123` or the MP3 library.
- **Standard:** Always resolve `project_root` from `pieces/locations/location_kvp`. Never trust `.` or `../`.

## 3. CREATIVE DESIGN DECISIONS

1.  **Dynamic Method Dispatch:** Instead of a static menu, we use the library scan to generate the `METHOD` section of the PDL. This makes the UI entirely data-driven. If the user adds a song to the folder, it appears on the next render.
2.  **String-Based Commands:** By using `onClick="MP3:PLAY:filename.mp3"`, we avoid index-based math. The command carries the data it needs. This is far more robust than "Button 1 = Song 1."
3.  **Spacebar for Zen:** Implementing `32` (Space) as a hardcoded toggle for pause/resume in the manager provides a "native app" feel within a terminal environment.
4.  **Shadow Xlector Sync:** Even in a menu app, we keep `active_target_id` pointed at the `mp3_player` piece. This ensures that `${piece_methods}` always reflects the music controls, even if the user is navigating other UI elements.

## 4. SUMMARY FOR FUTURE AGENTS
If you are building a list-based app (File browser, Jukebox, Item Shop):
1.  **Copy the `cyoa-engine` Manager.**
2.  **Generate your own PDL.**
3.  **Whitelist your prefix in the Parser.**
4.  **Use `inject_command` for Enter key support.**

*"If the UI feels dead, check your pulses (frame_changed.txt). If the input feels dead, check your injection format."*
