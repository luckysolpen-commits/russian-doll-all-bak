# Fuzzpet Bugs - March 7 Fix Session

## ✅ Bug #1: Selector Not Showing Automatically - FIXED

**Issue:** The selector cursor was not visible on screen at startup.

**Fix Applied:** Changed `selector_active` initialization from `0` to `1` in `fuzzpet/plugins/fuzzpet.c`.

---

## ✅ Bug #2: Clock Mode Not Defaulting to Turn-Based - FIXED

**Issue:** The fuzzpet clock mode was defaulting to "auto" instead of "turn" mode.

**Fix Applied:** Changed all default initializations from "auto" to "turn" in:
- `run_chtpm.sh`
- `clock_daemon/plugins/clock_daemon.c`
- `clock/plugins/increment_turn.c`
- `clock/plugins/increment_time.c`

---

## ✅ Bug #3: OS Main Menu Updating on Every Clock Tick - FIXED

**Issue:** The OS main menu was re-rendering on every clock tick (every second).

**Fix Applied:** 
1. Changed the fuzzpet clock plugins (`increment_turn.c` and `increment_time.c`) to hit `pieces/apps/fuzzpet_app/fuzzpet/view_changed.txt` instead of the global `master_ledger/frame_changed.txt`.
2. Made the system `clock_daemon` silent - it no longer hits the global frame marker. The daemon now only updates state silently; apps that need clock updates poll the clock_daemon state as needed.
