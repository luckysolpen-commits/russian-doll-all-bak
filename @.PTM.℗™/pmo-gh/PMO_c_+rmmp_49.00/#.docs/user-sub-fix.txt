## Executive Summary

This document provides the exact implementation pattern for user authentication dynamic submenus, following the TPM (True Piece Method) architecture demonstrated by working references (fuzz-op, os.chtpm).

**Sea Shell Philosophy:** 
The core functionality of CHTPM (parsing, jumping, navigation) represents the "strong inside" of the shell. We preserve its behavior (e.g., `do_jump` only moves focus). We strengthen the "weak new growth" (App-specific pathing, digit bounds) without breaking the core.

## Corrected Parser Implementation

### 1. Digit Navigation Bound Fix
**File:** `pieces/chtpm/plugins/chtpm_parser.c`

Instead of `count_projects()`, `process_key()` must use `MAX_ELEMENTS` (or `element_count`) as the upper bound for digit accumulation. This ensures navigation works for App layouts with many UI elements.

### 2. Navigation Integrity (The "do_jump" Rule)
`do_jump()` MUST remain focus-only. Do NOT add execution logic (`send_command`) to it.
```c
bool do_jump(int target_num) { 
    int cn = 0; 
    for (int i = 0; i < element_count; i++) { 
        if (is_navigable(i)) { 
            cn++; 
            if (cn == target_num) { 
                focus_index = i; 
                return true; 
            } 
        } 
    } 
    return false; 
}
```
**Mandate:** Execution remains triggered by Enter or explicit routing.

## Expected Behavior

1. **Focus**: Pressing digits (1, 2, 3...) moves the focus `[>]` to the corresponding button.
2. **Execution**: Pressing Enter on a focused button executes its `onClick` or `href`.
3. **Submenu**: Pressing Enter on "login" changes `active_target_id` to `user_login`, parser reloads methods.
