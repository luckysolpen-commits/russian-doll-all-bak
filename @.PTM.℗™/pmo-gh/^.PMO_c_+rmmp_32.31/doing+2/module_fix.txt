# MODULE_FIX: Parser Not Launching fuzzpet_v2_module
**Date:** 2026-03-12 00:15
**Issue:** fuzzpet_v2_module.+x is NOT being launched by parser
**Symptom:** Module debug.txt is empty, module not in process list

---

## THE PROBLEM

The layout `fuzzpet_v2.chtpm` has:
```xml
<module>projects/fuzzpet_v2/manager/+x/fuzzpet_v2_module.+x</module>
```

But the module never starts. Parser should launch it like a `<script>` tag in HTML.

---

## HOW MODULE LAUNCH SHOULD WORK

### Step 1: Parser Reads Layout
```
chtpm_parser.c parses fuzzpet_v2.chtpm
    ↓
Finds <module> tag
    ↓
Calls launch_module("projects/fuzzpet_v2/manager/+x/fuzzpet_v2_module.+x")
```

### Step 2: launch_module() Executes
```c
void launch_module(const char* path) {
    // Fork and exec the module binary
    pid_t p = fork();
    if (p == 0) {
        execl(path, path, NULL);
        exit(1);
    }
    current_module_pid = p;
}
```

### Step 3: Module Runs
```
fuzzpet_v2_module.+x starts
    ↓
Writes to debug.txt: "[00:15:00] V2 MODULE START"
    ↓
Polls keyboard/history.txt for keys
```

```c

    FILE *dbg = fopen("debug.txt", "a");
    if (dbg) {
        time_t t; struct tm *tm;
        time(&t); tm = localtime(&t);
        fprintf(dbg, "[%02d:%02d:%02d] PARSER: launch_module(%s)\n",
                tm->tm_hour, tm->tm_min, tm->tm_sec, launch_str);
        fclose(dbg);
```

## COMMON FAILURES

| Failure | Symptom | Fix |
|---------|---------|-----|
| Parser doesn't parse `<module>` | No "launch_module" in debug | Check parser tag handling |
| Module path wrong | "No such file" error | Verify path in layout |
| Module crashes on start | Process dies immediately | Check stderr, resolve_root() |
| Module not compiled | Binary missing | Run compile_all.sh |
| Wrong working directory | Can't find files | Check chdir() in launch |

## THE FIX

Once we identify the bottleneck, apply the fix:

### If Parser Doesn't Parse `<module>`:
Fix parser's tag handling in `chtpm_parser.c`

### If Module Path Wrong:
Update layout or make path absolute

### If Module Crashes:
Add more debug to find crash point

### If Working Directory Wrong:
Ensure launch_module() does chdir(project_root)
