# FUZZPET REFACTOR PLAN - M11
Goal: Migrate Fuzzpet from monolithic C app to Op-Driven content running on Player V2.

## 0. PRESERVATION & PLACEMENT
- [ ] Preserve legacy `pieces/apps/fuzzpet_app/` for reference.
- [ ] Placement: Fuzzpet V2 will be the **4th project** in the Player project list.
- [ ] Hotkeys:
  - **WASD**: Move Fuzzpet (consumes Energy)
  - **Key 1**: Feed (reduces Hunger)
  - **Key 2**: Play (increases Happiness)
  - **Key 3**: Sleep (restores Energy)
  - **Key 4**: Toggle Emoji/ASCII mode

## 1. PROJECT SETUP
- [ ] Create `projects/fuzzpet_v2/` directory structure.
- [ ] Initialize `projects/fuzzpet_v2/project.pdl`.
- [ ] Create Fuzzpet Piece DNA: `projects/fuzzpet_v2/pieces/fuzzpet/piece.pdl`.
- [ ] Initialize Fuzzpet State: `projects/fuzzpet_v2/pieces/fuzzpet/state.txt`.

## 2. SHARED OPS EXTRACTION (The APIs)
Extracted from legacy `fuzzpet.c`:
- [ ] `move_player.+x <piece_id> <dir>`
  - Handles 'T' (Treasure) collision: happiness += 10.
  - Updates `projects/{id}/pieces/{piece_id}/state.txt`.
- [ ] `stat_decay.+x <piece_id>`
  - Action: `hunger++`, `energy--`.
  - Logic: Clamp hunger [0-100], energy [0-100].
- [ ] `fuzzpet_action.+x <piece_id> <action>`
  - Actions: `feed` (hunger-20), `play` (happiness+10), `sleep` (energy+30).
  - Updates: Response ledger with result strings.
- [ ] `render_map.+x <project_id> <map_id>`
  - Compose view from terrain + all entities in `pieces/`.
  - Display Pet Status line: `H: ${hunger} | E: ${energy} | L: ${level}`.

## 3. PAL SCRIPTING (The Brain)
- [ ] Create `projects/fuzzpet_v2/scripts/game_loop.asm`.
- [ ] Map keys 1, 2, 3 to `fuzzpet_action` op calls.
- [ ] Map WASD to `move_player` op calls.
- [ ] Trigger `stat_decay` on move or turn end.

## 4. INTEGRATION
- [ ] Update `pieces/chtpm/layouts/os.chtpm` to launch Fuzzpet V2 project.
- [ ] Verify `player_manager.c` loads the correct project state.

## 5. VALIDATION
- [ ] KPI: Fuzzpet V2 renders via shared `render_map`.
- [ ] KPI: Feeding pet reduces hunger state in `pieces/fuzzpet/state.txt`.
- [ ] KPI: Movement logic consumes energy.
- [ ] KPI: All components compile without warnings (Compile Clean standard).
