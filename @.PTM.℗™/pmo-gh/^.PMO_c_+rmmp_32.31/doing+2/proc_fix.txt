# PROC_FIX: KEY DEBUG IN CONTROL MAP
**Date:** 2026-03-11 23:30
**Issue:** [KEY] shows "None" in interact mode, should show last key pressed
**Root Cause:** playrm_module.c overwrites last_key after PAL script runs
**Fix Location:** pieces/apps/playrm/plugins/playrm_module.c line 238
**Status:** ✅ FIX APPLIED

---

## ROOT CAUSE

In `playrm_module.c` for fuzzpet_v2 (lines 229-239):

```c
// Line 217-226: Key processed, last_key_str set, saved to disk ✓
if (key >= 32 && key <= 126) snprintf(last_key_str, ...);
save_engine_state();  // Saves last_key=ENTER

// Line 229-237: PAL script runs
system(prisc_cmd);

// Line 238: BUG - reloads STALE state from disk!
load_engine_state();  // ← Overwrites last_key_str with last_key=None from disk!
```

**Result:** The `last_key` that was set before PAL runs gets overwritten by the stale value from disk.

---

## THE FIX

**File:** `pieces/apps/playrm/plugins/playrm_module.c`

**Removed line 238** (`load_engine_state();`):

```c
// BEFORE (broken):
if (prisc_cmd) {
    system(prisc_cmd);
    free(prisc_cmd);
}
load_engine_state();  // ← REMOVED

// AFTER (fixed):
if (prisc_cmd) {
    system(prisc_cmd);
    free(prisc_cmd);
}
// FIX: Removed load_engine_state() - it was overwriting last_key!
// PAL script already calls render_map.+x and hit_frame
```
