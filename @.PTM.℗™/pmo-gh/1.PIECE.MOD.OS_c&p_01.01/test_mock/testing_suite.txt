# testing_suite.txt - Autonomous PMO Pipeline Verification

This suite allows for full system testing without human interaction by manually injecting key codes and auditing the resulting frame buffer.

---

## 1. THE "GHOST HAND" PROTOCOL (Autonomous Input)

### 1.1 Test Case: OS Start and Nav-to-App
*   **Action:** Start `run_pmo.py`.
*   **Verification:** `current_frame.txt` must contain "CHTPM+OS Main Menu".
*   **Action:** `echo 1003 >> keyboard/history.txt` (Down).
*   **Verification:** `focus_index` in `chtpm/state.txt` should increment.
*   **Action:** `echo 13 >> keyboard/history.txt` (Enter).
*   **Verification:** `current_layout` in `chtpm/state.txt` should be `main.chtpm`.

### 1.2 Test Case: Digit Jumping
*   **Action:** In OS Menu, `echo 2 >> keyboard/history.txt`.
*   **Verification:** `focus_index` should point to index 2 (Launch FuzzPet).
*   **Constraint:** System MUST NOT transition to FuzzPet dashboard yet.

## 2. THE RENDERING AUDIT (Aesthetics)

### 2.1 Aesthetic Invariants
*   **Clean Output:** No string starting with `<` should exist in `current_frame.txt`.
*   **Tight Vertical:** Frame lines should match (Elements + Header/Footer + Pulse Margin).
*   **Box Sovereignty:** Every frame line must be wrapped in ║ or ╠ or ╚ or the OS header.
