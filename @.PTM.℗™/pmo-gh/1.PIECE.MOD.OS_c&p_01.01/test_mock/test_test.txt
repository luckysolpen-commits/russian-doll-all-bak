## ROOT CAUSE ANALYSIS

### Problem 1: Parser is rendering RAW XML tags

The `<menu>` and `<text>` tags appearing in output means the parser's
`parse_layout_elements()` is NOT consuming these tags properly.

### Problem 2: `${game_map}` contains multi-line content with embedded tags

The fuzzpet `view.txt` contains:
```
║  ##########                                                ║
...
╠═══════════════════════════════════════════════════════════╣
║  [FUZZPET]: ...                                           ║
```

When this is substituted into `${game_map}`, the parser sees newlines
and the content spans multiple "lines" in the layout.

### Problem 3: The layout file has `${game_map}` as a LABEL, not standalone

In `main.chtpm`:
```xml
<text label="${game_map}" /><br/>
```

This means the ENTIRE game map content becomes the `label` attribute of
a text element. When rendered, it should just output the content.

BUT - the view.txt content itself contains box-drawing characters that
look like layout structure, confusing the parser.
