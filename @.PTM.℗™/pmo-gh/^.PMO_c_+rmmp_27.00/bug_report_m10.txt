BUG REPORT - 2026-03-10
Module: editor.chtpm / chtpm_parser.c
Status: CRITICAL UI CORRUPTION

SYMPTOM:
The RMMP Editor view shows all four tabs (MAP, PIECES, DB, PROJECT) mashed together in a single frame, making the interface unreadable.

ROOT CAUSE:
1. Unsupported Feature: I attempted to use <if condition="..."> tags in editor.chtpm to handle tab switching.
2. Parser Limitation: The 'chtpm_parser.c' DOES NOT implement any logic for <if> tags. It simply ignores the tags themselves but renders all text/buttons contained within them.
3. Result: Every tab's content is rendered unconditionally every frame.

FIX STRATEGY:
1. Remove the <if> tags from editor.chtpm.
2. Replace the multi-tab area with a single variable: ${editor_tab_content}.
3. Update 'editor_module.c' to dynamically generate the XML/CHTPM markup for the active tab and export it via state.txt.
4. This keeps the parser simple and the UI clean.
