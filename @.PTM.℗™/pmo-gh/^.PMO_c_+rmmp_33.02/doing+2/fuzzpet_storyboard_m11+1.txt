## Chapter 1: The Main Menu

### 1.1. Description

The application starts in a command-line interface (CLI) environment titled "CHTPM+OS CLI v2.0". This serves as the main menu and entry point to the FuzzPet application. The screen is divided into four main sections: a header, a main content area, a status bar, and a command input area.

### 1.2. UI Elements

*   **Header:** Displays the title "C H T P M + O S C L I v2.0".
*   **Main Content Area:** Contains a list of selectable options. The currently selected option is indicated by `[>]`, while unselected options are marked with `[ ]`. The options are:
    *   1. Help
    *   2. Launch FuzzPet
    *   3. Status
    *   4. List Processes
    *   5. Settings
    *   6. Toggle Frame History
    *   7. GL-OS
*   **Status Bar:** Displays information about running applications and system time.
*   **Command Input Area:** A prompt labeled "Nav >" where the user can input commands.

### 1.3. Functionality

*   The user can navigate through the options by inputting the corresponding number.
*   Selecting "2. [Launch FuzzPet]" transitions the user to the FuzzPet Launcher.

## Chapter 2: The FuzzPet Launcher

### 2.1. Description

After launching FuzzPet from the main menu, the user is presented with the "FUZZPET LAUNCHER". This screen allows the user to start a new game or continue a previous one.

### 2.2. UI Elements

*   **Header:** Displays the title "F U Z Z P E T L A U N C H E R".
*   **Main Content Area:** A list of options for starting the game:
    *   1. New Game (Reset Pet)
    *   2. Load Game (Continue)
    *   3. Back to Main Menu
*   **Command Input Area:** A prompt labeled "Nav >" for user input.

### 2.3. Functionality

*   The user can select an option by entering its number.
*   Selecting "1. [New Game (Reset Pet)]" starts a new FuzzPet game and takes the user to the FuzzPet Dashboard.
*   Selecting "2. [Load Game (Continue)]" would presumably load a saved game state.
*   Selecting "3. [Back to Main Menu]" returns the user to the CHTPM+OS Main Menu.

## Chapter 3: The FuzzPet Dashboard (Selector Mode)

### 3.1. Description

This is the primary interface for the FuzzPet game when the user is in "Selector Mode". The screen is divided into several panels: a header, a pet status panel, a game map, a response panel, and an actions panel.

### 3.2 UI Elements

*   **Header:** Displays the title "F U Z Z P E T D A S H B O A R D" (or project-specific title).
*   **Pet Status Panel:** Shows the current pet's name, status, and vital statistics.
*   **Game Map:** ASCII representation of the world.
*   **Debug & Response Panel:** 
    *   Displays feedback messages ("Yum yum!").
    *   **Debug Line:** Shows `[ACTIVE]: <target_id>` and `[KEY]: <last_keypress>` to verify input routing and focus.
*   **Actions Panel:** List of available entity-dependent methods.


### 3.3. Functionality

*   The user can move the selector around the map using directional inputs (e.g., UP, RIGHT).
*   The "Control Map" option allows the user to directly interact with the map and move the selector. When this option is active, the prompt at the bottom of the screen changes to "Active [^]: (ESC to exit)".
*   The "Scan" action lists the entities present on the map.
*   The user can switch from Selector Mode to Pet Mode by moving the selector over a pet and presumably pressing an "select" or "enter" key.

## Chapter 4: The FuzzPet Dashboard (Pet Mode)

### 4.1. Description

This screen is displayed when the user has selected a pet to interact with. The layout is similar to the Selector Mode dashboard, but the available actions and the information in the status panel are specific to the selected pet.

### 4.2. UI Elements

*   **Header:** Displays the title "F U Z Z P E T D A S H B O A R D".
*   **Pet Status Panel:** Shows the selected pet's name (e.g., "Fuzzball"), status ("active"), and vital statistics, which now have non-zero values (e.g., Hunger: 30/100, Happiness: 55/100).
*   **Game Map:** The same map as in Selector Mode, but the user now controls the pet directly, represented by '@'.
*   **Response Panel:** Displays feedback related to the pet's actions (e.g., "Yum yum!", "Wheee~ Best playtime ever!").
*   **Actions Panel:** A list of pet-specific actions:
    *   1. Control Map
    *   2. Feed
    *   3. Play
    *   4. Sleep
    *   5. Toggle_emoji
    *   6. Back to Selector
    *   7. Exit to Main Menu

### 4.3. Functionality

*   The user can control the selected pet's movement on the map when "Control Map" is active.
*   The user can interact with the pet by selecting actions like "Feed" and "Play", which affect the pet's stats and produce corresponding feedback messages.
*   Selecting "Back to Selector" returns the user to Selector Mode.

## Glossary

*   **ASCII:** American Standard Code for Information Interchange. A character encoding standard for electronic communication. In this context, it refers to the use of text characters to create a visual representation of the game world.
*   **CLI:** Command-Line Interface. A text-based user interface used to view and manage computer files and run programs.
*   **Dashboard:** A user interface that organizes and presents information in a way that is easy to read.
*   **Entity:** Any object in the game world, such as a pet, a resource, or the selector.
*   **FuzzPet:** The name of the virtual pet game.
*   **Selector:** A user-controlled entity used to navigate the game map and interact with other entities.
*   **Selector Mode:** The game state where the user controls the selector.
*   **Pet Mode:** The game state where the user directly controls a selected pet.
*   **UI:** User Interface. The means by which the user and a computer system interact.
