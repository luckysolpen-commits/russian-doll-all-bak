## Guiding Philosophy: TPM K.I.S.S.

> **"The original FuzzPet pipeline (`c+x_tpm_FUZPET+gl_12+PDLON/`) is the reference architecture. All new implementations must match its patterns."**

### The TPM K.I.S.S. Methodology

1. **Keep** - Use the exact file patterns from original FuzzPet.
2. **It** - Each piece owns its files; modules are pieces too.
3. **Simple** - File-based communication throughout (no pipes).
4. **Stay TPM-Compliant** - Parser, modules, and renderers communicate via files.

## Part 2: File Communication API

### 2.1 Module Input Queue: `pieces/fuzzpet/history.txt`
**Producer:** `chtpm_parser.+x` (via `inject_raw_key()`)  
**Consumer:** `fuzzpet.+x` module  
**Mandate:** Parser only injects keys here when an element with `onClick="INTERACT"` is active.

### 2.2 Frame Output: `pieces/display/current_frame.txt`
**Producer:** **Whichever process owns the input.**
*   In Navigation Mode: The Parser.
*   In INTERACT Mode: The Module.
**Consumer:** `renderer.+x`.

### 2.3 Frame Signal: `pieces/master_ledger/frame_changed.txt`
**Protocol:** Append "X\n" immediately after writing `current_frame.txt`. 
**Mandate:** Do not use `mtime`. Readers must monitor **File Size**.

## Part 3: Architecture Summary - Answered Questions

### Q1: Should parser compose frames at all when module is active?
**ANSWER: NO.**
**Rationale:** To prevent "UI Fighting" and flickering, only one process can write to the frame file at a time. When the user enters a game/canvas (`INTERACT` mode), the Module takes sole ownership of `current_frame.txt`. The Parser becomes a silent input relay until `ESC` is pressed.

### Q2: How should parser signal UI state changes to module?
**ANSWER: `history.txt` Injection.**
**Rationale:** The Parser calls `inject_raw_key(code)`, appending the raw integer to the module's private history. The module polls this file using `fseek` to detect and process the new keys.

## Part 4: Macro Folder Sovereignty

All application-specific pieces MUST be grouped in `pieces/apps/<app_name>/`.
*   **Good:** `pieces/apps/fuzzpet_app/world/map.txt`
*   **Bad:** `pieces/world/map.txt` (This creates scattering and ownership confusion).
