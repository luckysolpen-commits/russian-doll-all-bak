## 0. THE VIRGIN CONTEXT MANDATE
PMO (Piece-Module-OS) is a system where **The Filesystem is the Brain**. 
Code is a temporary ghost; the directory is the inhabitant. 

---

## 1. THE TRINITY HIERARCHY (P-M-O)

### 1.1 Priority I: THE PIECE (The Atom)
A Piece owns its own data. It has two state representations:
*   **The Master DNA (`.pdl`):** The absolute source of truth. Contains `STATE`, `METHODS`, and `RESPONSES`. Modified via `piece_manager` for a perfect audit trail.
*   **The Fast Mirror (`state.txt`):** A machine-readable `key=value` cache. Used for high-speed polling by Renderers and the OS.

### 1.2 Priority II: THE MODULE (The Logic Agent)
The Module is the brain. It MUST maintain the **Mirror Sync**:
1.  Calculate Logic.
2.  Update **Master DNA** (via `piece_manager set-state`) for the Ledger.
3.  Flush to **Fast Mirror** (`state.txt`) for the Renderer.
4.  Compose the Frame (if in INTERACT mode).

### 1.3 Priority III: THE OS / LAYOUT (The Viewer)
The OS (`chtpm_parser`) uses the **Fast Mirror** to populate `${variables}` in the UI without triggering expensive `piece_manager` calls.

---

## 2. THE KISS PIPELINE ( custody of the screen )

Every user action follows this linear chain:
1.  **CAPTURE:** `keyboard_input` -> `pieces/keyboard/history.txt`.
2.  **RELAY:** `chtpm_parser` reads history -> injects to `pieces/<module>/history.txt`.
3.  **LOGIC:** Module consumes key -> updates stats.
4.  **SYNC:** Module updates `.pdl` (Audit) AND `state.txt` (Mirror).
5.  **COMPOSE:** Module generates the ASCII frame directly.
6.  **SIGNAL:** Module hits the Marker (`frame_changed.txt`).
7.  **RENDER:** Renderer prints `current_frame.txt`.

---

## 3. INPUT & NAVIGATION (The Dual-Mode)

### 3.1 Navigation Mode (`active_index == -1`)
Parser is the "Writer". It moves focus indicators and updates the mirror variables in the frame.

### 3.2 Passthrough / INTERACT Mode
Module is the "Writer". 
*   **CRITICAL:** The Parser **STOPS** composing the frame. 
*   The Module **STARTS** composing the frame. 
*   If both write, the screen will flicker. Only the "Logic Owner" may be the "View Owner".

---

## 4. SIGNALING: THE MARKER-FILE PROTOCOL
Do not use `mtime`. Readers MUST monitor **File Size** of `frame_changed.txt`. 
*   Append "X\n" = Wake up Renderer.

---

## 5. MACRO FOLDERS
Group related pieces in `pieces/apps/<app_name>/`. 
*   **Sovereignty Rule:** Modules should not share "in-memory" state. If two apps need the same data, they must read it from the same Piece file on disk.

---

## 6. THE THREE-MANAGER PIPELINE
1.  **Piece Manager:** Operates on the `.pdl` (The DNA).
2.  **Methods Manager:** Locates `+x` binaries.
3.  **Event Manager:** Dispatches `RESPONSE` strings.
