# CHTPM v0.05 Refactor: Purging Hardcoded Commands
Date: 2026-03-05
Goal: Decouple chtpm_parser from app-specific logic (FuzzPet).

## 1. Research & Audit
- [ ] Audit all `.chtpm` layout files for hardcoded `onClick` actions (e.g., `FEED`, `SCAN`, `PLAY`).
- [ ] Audit `chtpm_parser.c`'s `handle_custom_command` function for all `strcmp` blocks.
- [ ] Verify `selector` control functionality (movement and interaction) to ensure no regressions.

## 2. Parser Refactor (The Theater)
- [ ] Implement generic `KEY:n` command in `handle_custom_command` to allow layout files to trigger raw key injections without parser-side mapping.
- [ ] Remove hardcoded blocks: `SCAN`, `COLLECT`, `PLACE`, `FEED`, `PLAY`, `SLEEP`, `CLOCK`.
- [ ] Ensure `BACK` remains or is generalized to a common key (e.g., `KEY:9`).

## 3. Layout Update (The UI)
- [ ] Update `fuzzpet_launcher.chtpm` to use generic triggers.
- [ ] Update `main.chtpm` / `os.chtpm` to ensure system buttons are decoupled.
- [ ] Update FuzzPet dashboard elements to use `onClick="KEY:2"`, `onClick="KEY:3"`, etc., moving the "meaning" of these keys entirely to the Manager.

## 4. Manager Refactor (The Stage)
- [ ] Update `fuzzpet_manager.c` to handle inputs more robustly.
- [ ] Ensure the Manager correctly interprets `KEY:2` as `FEED` for a pet but `SCAN` for a selector, based on `active_target_id`.
- [ ] (Optional/Phase 2) Implement "Method Mapping" where the Manager looks up the `.pdl` to execute named methods.

## 5. Validation
- [ ] Compile all updated components using `compile_all.sh`.
- [ ] Run `run_chtpm.sh` and verify:
    - [ ] Selector moves correctly.
    - [ ] Selecting a pet works.
    - [ ] Buttons (Feed, Play, etc.) still fire correctly through the new decoupled path.
    - [ ] Menu navigation remains intact.
