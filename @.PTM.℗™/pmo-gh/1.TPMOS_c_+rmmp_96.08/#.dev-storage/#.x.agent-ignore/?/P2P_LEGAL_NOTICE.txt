SECTION 6: DEVELOPER NOTES
================================================================================

**For Future Developers:**

```
DESIGN PRINCIPLES:

1. Fun-first, always.
   - If a technical decision makes the game less fun, reconsider.
   - Refunds are cheap (it's fake coins!).
   - User frustration is expensive.

2. Multiple wallets are a feature, not a bug.
   - Users should be able to create unlimited wallets.
   - Switching should be instant and free.
   - All wallets are linked to the user profile.

3. Security is "Casual Fun" level.
   - SHA256(password + salt) is fine.
   - Support can recover/reset anything.
   - No need for 2FA, hardware wallets, etc.
   - This is a GAME, not a bank.

4. Refunds are automatic and generous.
   - Log everything to master_ledger.txt.
   - Support tools should make refunds easy.
   - Never argue with a user about lost coins.
   - Just refund and move on.

5. Transparency.
   - All transactions logged.
   - Users can see their history.
   - Support can investigate any issue.
```
