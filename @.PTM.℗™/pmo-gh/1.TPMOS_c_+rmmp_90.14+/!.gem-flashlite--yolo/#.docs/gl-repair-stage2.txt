# Fuzz-Op-GL Repair Analysis & Stage 2 Plan

**Date:** April 24, 2026
**Subject:** Resolving Camera Mode Glitches and Input Routing Anomalies

## 1. Identified Issues

### A. Camera Mode Glitches (Numeric Keys 1-4)
*   **Symptom:** Selecting camera modes 1-4 via keyboard causes flickering or the mode being immediately overwritten.
*   **Root Cause:** Conflict of Sovereignty. Both the Host (`gl_desktop.c`) and the Manager (`fuzz-op-gl_manager.c`) are attempting to manage the `camera_mode` state simultaneously. 
    *   The Host updates its local `win->camera_mode` immediately upon key press.
    *   The Host also injects the key press into the project history.
    *   The Manager reads the history, updates its internal `camera_mode`, and writes it to `session/state.txt`.
    *   The Host reloads `session/state.txt` on the next pulse, overwriting its local state with the Manager's state (which might be slightly delayed or out of sync).

### B. WASD Controlling Xelector instead of Camera
*   **Symptom:** In GL-OS, WASD keys move the Xelector piece instead of the 3D camera.
*   **Root Cause:**
    1.  **Manager Routing:** The `fuzz-op-gl_manager.c` routes WASD to entity movement via `route_input()` when `is_map_control` is 0. If the system is out of sync and the manager thinks it's in "Menu Mode" while the user is in "Map Mode", WASD moves the active target (default: `xlector`).
    2.  **Generic Op Logic:** The `move_entity.c` Op specifically handles WASD as directional aliases. In a GL context, WASD should be reserved for the Host's camera logic or the Manager's camera state, not generic piece movement.

### C. Arrows moving "Shadow Xelector" vs. "Wireframe Box"
*   **Symptom:** Arrow keys move a logical cursor (shadow) but the 3D wireframe box (drawn by Host) is decoupled or doesn't move.
*   **Root Cause:** 
    *   The Host draws the wireframe box using `win->xelector_pos`.
    *   `win->xelector_pos` is synced from `session/state.txt` variables `selector_pos_x` and `selector_pos_y`.
    *   **Mismatch & Confusion:** There is a naming inconsistency between `selector` and `xelector`.
        *   **Selector:** Reserved for **Navigation Control** (UI menu indexing).
        *   **Xelector:** Reserved for the **Wireframe Piece** (3D map cursor).
    *   The Manager updates variables in `save_manager_state()` by reading the `xlector` piece's state but uses `selector_pos_x` as the key. This naming drift causes confusion and decoupling when some logic expects a "selector" piece that doesn't exist or is for UI nav only.

## 2. Repair Strategy (Stage 2)

### Goal 1: Establish Single Source of Truth for Camera
*   **Host Change:** `gl_desktop.c` should **NOT** update `win->camera_mode` locally for `gltpm_app`. It should ONLY inject the key press into history and wait for the Manager to update the state.
*   **Manager Change:** Manager remains the authority. Ensure it pulses the `frame_changed.txt` immediately after updating camera state.

### Goal 2: Decouple WASD and Arrow Keys (GL-Standard)
*   **Caveat Documentation:** 
    *   **WASD:** Reserved for **Host-side Camera Movement** (Camera Fly/ISO-pan). These should NOT be injected into history as generic keys if we want smooth movement, OR they should be injected specifically for "CAMERA_MOVE" events.
    *   **Arrow Keys:** Reserved for **Xelector/Piece Movement**. These are injected as `KEY_PRESSED: 1000-1003` and handled by the Manager to move the `xlector` Piece.
*   **Host Change:** Modify `keyboard()` in `gl_desktop.c` to handle WASD for camera movement directly (as it does for `project_mirror` type) and stop injecting them as generic keys for modular apps.

### Goal 3: Unified Xelector Piece & UI Consistency
*   **Audit:** Ensure all Ops and the Manager refer to the piece ID `xlector` and the state file `projects/fuzz-op-gl/pieces/xlector/state.txt`.
*   **HUD Update:** Rename the coordinate display in `gl_desktop.c` from `"SEL:"` to `"XEL:"` to distinguish it from navigation selection.
*   **Cleanup:** Remove any legacy references to `selector` piece in the modular project.

## 3. Implementation Plan

1.  **Modify `gl_desktop.c`:**
    *   Update `keyboard()` routing for `gltpm_app`.
    *   Add WASD/ZXQE camera movement logic to `gltpm_app` block (mirroring `project_mirror` logic).
    *   Ensure numeric keys 1-4 only inject to history, don't set local state.
    *   Rename HUD HUD label `"SEL:"` to `"XEL:"`.
2.  **Modify `fuzz-op-gl_manager.c`:**
    *   Ensure `is_map_control` is handled as absolute state (Set to 1 on INTERACT, 0 on ESC), not just toggled.
    *   Refine `gltpm_input_thread` to ignore WASD if they are handled by Host.
    *   Standardize all state variable keys to use `xelector_pos_x` instead of `selector_pos_x` (Update `save_manager_state` and Host reading logic).
3.  **Verify Piece Alignment:**
    *   Ensure `move_entity.+x` is the only movement Op used for Xelector.
    *   Check `session/state.txt` mappings in Manager.
