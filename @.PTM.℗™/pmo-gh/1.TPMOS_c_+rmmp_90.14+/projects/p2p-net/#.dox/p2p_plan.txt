P2P-NET STRATEGIC PLAN (M21)
=============================

1. PROJECT GOAL
---------------
Implement a TPM-Compliant P2P Network featuring Blockchain, NFTs (Items), Trading, and Auctions.

2. ARCHITECTURE (PROJECT-BASED)
-------------------------------
Path: `projects/p2p-net/`

- manager/p2p_manager.+x: The brain (inspired by SHIP/main.c).
- pieces/blockchain/chain.txt: The global ledger of verified blocks.
- pieces/wallet/state.txt: Tracks balance, address, and login status.
- pieces/inventory/ (Piece collection): Each NFT is a separate Piece with hash-based identity.

3. CORE NETWORK FEATURES
-------------------------

A. PEER CONNECTION (TCP)
- Use standard TPM networking (127.0.0.1:8000-8010).
- Handshake: Hello -> Ack loop.
- Discovery: Peers broadcast active lists via `pieces/network/peers.txt`.

B. BLOCKCHAIN & MINING
- Each block contains a type (Manual, Send, Trade, Auction).
- Mining difficulty is set in `project.pdl`.
- Reward is automatically credited to the `wallet/state.txt` mirror.

C. THE NFT ECONOMY (NFT-OPS)
- Items have metadata: `attack`, `durability`, `rarity`.
- Hashing: `sha256(name + address + time)`.
- Auctions: Escrow logic where an item piece is moved to `projects/p2p-net/pieces/auctions/`.

4. UI LAYOUT (p2p-net.chtpm)
----------------------------
- Dashboard:
    - Wallet Address: ${wallet_address}
    - Balance: ${wallet_balance}
- Actions:
    [1] Peer List (Connect/Disconnect)
    [2] Mine Block (Gain TPM coins)
    [3] My Inventory (Manage NFTs)
    [4] Auction House (Active auctions)
    [5] (BLANK) - Infrastructure for P2P Trading.

5. INFRASTRUCTURE NOTES
-----------------------
- Uses OpenSSL (SHA256/RSA) for cryptographic sovereignity.
- Aligned with `2.c2c_tpm_1.0000]SHIP/main.c` networking logic.
