# MP3 Store Fix Documentation

## Root Cause Analysis (Hypothesized)

**Bug 1: Directory Expansion Not Working**
- **Symptom:** Pressing ENTER does not visually expand or collapse directory listings in the MP3 Store.
- **Hypothesis:** The `mp3_store_manager.c` correctly generates the `directory_listing` string and writes it to `state.txt`. However, `chtpm_parser.c` might be failing to:
    1. Correctly load the `directory_listing` variable from `pieces/apps/player_app/manager/state.txt` due to formatting issues or parsing errors.
    2. Substitute the `${directory_listing}` variable correctly in the final frame rendering.

**Bug 2: Keypress Feedback Not Displayed**
- **Symptom:** No visual feedback is provided for key presses in the MP3 Store UI.
- **Hypothesis:** Similar to the directory listing, `mp3_store_manager.c` writes the `last_key_str` to `state.txt`. However, `chtpm_parser.c` may not be correctly loading or substituting the `${last_key}` variable from the state file for display in the layout. The `fuzz-op` project demonstrates this mechanism is functional in other modern layouts.

**Fix for Bug 2 (Keypress Feedback):**
1.  **Analyze Debug Output:** Examine the `DEBUG: Loaded last_key: ...` output from `load_vars()` in `chtpm_parser.c` to confirm if `last_key` is being loaded correctly from `state.txt`.

3.  **Ensure State Update:** Confirm that `mp3_store_manager.c` is correctly updating and writing `last_key_str` to `state.txt` in a parsable format.
4.  **Ensure Variable Substitution:** If `last_key` is loaded correctly but not displayed, investigate the rendering of the `${last_key}` variable in the layout within `compose_frame()`.
