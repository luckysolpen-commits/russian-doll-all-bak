================================================================================
HOW YOU'LL TEST (GUI - WEEK 3+)
================================================================================

STEP 1: LAUNCH P2P-NET
──────────────────────
From main menu or loader:
  → Select "p2p-net"
  → Layout: p2p-net.chtpm loads

STEP 2: CONFIGURE SUBNET
────────────────────────
Press: [1]
Input: subnet_name = "chat"
Input: network_mode = "broadcast"
Input: connection_mode = "continuous"
Input: port = "8001"
Expected: "Configured subnet 'chat'"

STEP 3: JOIN SUBNET
───────────────────
Press: [2]
Input: subnet_name = "chat"
Expected: "Joined subnet 'chat'"
         Peer list shows other connected peers

STEP 4: SEND MESSAGE
────────────────────
Press: [5]
Input: peer_wallet = "abc123..."
Input: message = "Hello!"
Input: subnet = "chat"
Expected: "Message sent to abc123..."
         Chat log shows your message

STEP 5: VIEW CHAT LOG
─────────────────────
Press: [8] or navigate to chat section
Expected: See all messages for selected subnet
         Messages from multiple peers visible

STEP 6: MULTI-NETWORK TEST
──────────────────────────
1. Join "chat" subnet (Step 3)
2. Join "general" subnet (Step 3 again)
3. Check peer list - should show peers from BOTH subnets
4. Send message on "chat"
5. Send message on "general"
6. Switch subnet view - messages should be separate!

================================================================================
WHAT'S HAPPENING BEHIND THE SCENES
================================================================================

When you press buttons in the GUI:

Button [1] → Calls: configure_subnet op
Button [2] → Calls: broadcast_join op
Button [3] → Calls: list_peers op
Button [4] → Calls: connect_peer op
Button [5] → Calls: send_message op
Button [7] → Calls: broadcast_leave op

The p2p_manager.c handles:
- Reading your button presses
- Calling the right op
- Updating the display
- Managing peer connections in background
