Minimalist Philosophy: RISC-V designers omitted nand and nor because they can be easily emulated, keeping the base hardware simple and reducing the instruction count for the core ISA.
---------------------------
To build a basic RV32I (32-bit Integer) emulator, you should focus on the subset of instructions that allow for arithmetic, control flow (loops), and memory access. While the full base set has about 40 instructions, you can run simple programs with these core 10: [1, 2, 3, 4, 5] 
1. Essential Arithmetic & Logic

* ADDI (Add Immediate): The "Swiss Army Knife" of RISC-V. Used to load small constants into registers (e.g., addi x1, x0, 10) and for the MV (Move) pseudoinstruction.
* ADD / SUB: Basic register-to-register addition and subtraction for standard calculations.
* LUI (Load Upper Immediate): Critical for loading large 32-bit constants into registers. It places a 20-bit immediate in the top 20 bits of the destination register. [6, 7, 8, 9, 10] 

2. Control Flow (Loops and Jumps)

* BNE (Branch if Not Equal): Essential for implementing loops (e.g., "loop until counter reaches zero").
* BEQ (Branch if Equal): The counterpart to BNE, used for if statements and basic logic.
* JAL (Jump and Link): Used for jumping to a specific address and saving the return address, making function calls possible.
* JALR (Jump and Link Register): Necessary for returning from functions (used with the ra register) and for jumping to addresses stored in registers. [7, 9, 11, 12, 13] 

3. Memory Access

* LW (Load Word): Reads a 4-byte value from memory into a register.
* SW (Store Word): Writes a 4-byte value from a register into memory. [7, 11, 14, 15, 16] 

4. System / Execution

* ECALL (Environment Call): While not strictly for "logic," it is how your emulated program interacts with the outside world (e.g., printing a character or exiting the program). [11, 17, 18, 19] 

Summary Table for Implementation

| Instruction [3, 9, 20, 21, 22] | Type | Opcode (bits 6-0) | Purpose |
|---|---|---|---|
| ADDI | I-Type | 0010011 | Math with constants, loading small numbers |
| ADD/SUB | R-Type | 0110011 | Core arithmetic between registers |
| LUI | U-Type | 0110111 | Loading large 32-bit values |
| LW | I-Type | 0000011 | Reading data from RAM |
| SW | S-Type | 0100011 | Writing data to RAM |
| BEQ/BNE | B-Type | 1100011 | Loop and conditional logic |
| JAL | J-Type | 1101111 | Function calls and absolute jumps |

✦ To create a minimal 2-pass RISC emulator, I will implement a system that supports a
  subset of the RISC-V instruction set.


  Implementation Strategy
   1. Architecture: 16 registers (x0-x15), where x0 is hardwired to 0.
   2. Instruction Set:
       * addi rd, rs1, imm: R[rd] = R[rs1] + imm
       * beq rs1, rs2, label: Branch if equal.
       * lw rd, imm(rs1): Load word from memory.
       * sw rs2, imm(rs1): Store word to memory.
       * jalr rd, rs1, imm: Jump and link register (Standard for function calls/returns).
       * out rs1: Custom instruction to print register value to CLI/File.
       * halt: Stop execution.
   3. Two Passes:
       * Pass 1: Scans the assembly file for labels (e.g., loop:) and maps them to
         instruction indices.
       * Pass 2: Parses instructions into an internal executable format, resolving label
         names to memory offsets.
