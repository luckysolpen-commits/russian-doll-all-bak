# 29-report.txt - TPMOS Emergent Issues Report
Date: 2026-04-29
Status: ANALYSIS COMPLETE

## 1. Loader Cleanup (ASCII)
- **Findings:** `pieces/apps/playrm/loader/loader_module.c` dynamically populates `loader.pdl` by scanning the `projects/` directory. Currently, it includes all projects with a `project.pdl`, including GL-only ones like `aow-2d`, `op-ed-gl`, and `piececraft-3d`.
- **Plan:** Modify `loader_module.c` to skip `aow-2d` (and other GL projects) during the scan to prevent "Theater Pollution".

## 2. GL-OS Decoupling
- **Findings:** `pieces/apps/gl_os/gl_os_projects.kvp` already exists with the necessary mappings. However, `pieces/apps/gl_os/plugins/gl_desktop.c` still uses hardcoded strings in its menu navigation and execution logic.
- **Plan:** Implement `load_gl_projects()` in `gl_desktop.c` to read the KVP file. Update `execute_option` and the menu population logic to use this dynamic list.

## 3. 2D Rendering Enforcement (aow-2d)
- **Findings:** `gl_desktop.c` handles `project_mirror` windows with a generic 3D view. `aow-2d` is intended to be a top-down 2D experience.
- **Plan:** Add an override in `draw_window` (specifically for `project_mirror` type) that forces `cam_rot` to `(-90, 0, 0)` and `camera_mode` to a fixed state when `project_id` matches `aow-2d`.

## 4. Bible Update
- **Findings:** `!.gem-flashlite--yolo/!.evo&obs/!.TPMOS_ONBORD_BIBLE_2.md` has a Top 5 Pitfalls section.
- **Plan:** Add "Pitfall #6: Project Type Pollution" to document the requirement of keeping GL and ASCII loaders separate.

## Next Steps
- Modify `loader_module.c` to skip `aow-2d`.
- Update `gl_desktop.c` for KVP loading and dynamic menu.
- Apply `aow-2d` 2D camera override in `gl_desktop.c`.
- Update `!.TPMOS_ONBORD_BIBLE_2.md`.
