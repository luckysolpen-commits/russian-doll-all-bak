# HDLB0 RISC-V Architecture Specifications

## Stage 3: 8-bit RISC-V Implementation

### Objective
Extend the 2-bit RISC-V processor to an 8-bit processor while maintaining backward compatibility.

### Architecture Overview
- Word size: 8 bits
- Address size: 8 bits
- Register file: 32 registers (8 bits each) - following standard RISC-V
- Instruction set: Extended RISC-V subset

### Register File
- 32 general-purpose registers (x0-x31)
- x0 hardwired to 0
- Each register: 8 bits wide

### Instruction Set Architecture (ISA)
#### Instruction Formats (8-bit implementation)
1. R-type (Register)
   - [7:5] opcode | [4:2] rs1 | [1:0] rs2 | [7:5] rd | [4:2] funct3 | [1:0] funct7
2. I-type (Immediate)
   - [7:5] opcode | [4:2] rs1 | [1:0] immediate | [7:5] rd
3. S-type (Store)
   - [7:5] opcode | [4:2] rs1 | [1:0] rs2 | [4:2] immediate | [1:0] funct3
4. B-type (Branch)
   - [7:5] opcode | [4:2] rs1 | [1:0] rs2 | [4:2] immediate | [1:0] funct3
5. U-type (Upper Immediate)
   - [7:5] opcode | [4:0] immediate | [7:5] rd
6. J-type (Jump)
   - [7:5] opcode | [4:0] immediate | [7:5] rd

#### Supported Instructions (RV32I subset adapted for 8-bit)
1. Integer Register-Immediate Instructions
   - ADDI: Add Immediate
   - ANDI: AND Immediate
   - ORI: OR Immediate
   - XORI: XOR Immediate
   - SLTI: Set if Less Than Immediate
   - SLTIU: Set if Less Than Immediate Unsigned

2. Integer Register-Register Operations
   - ADD: Add
   - SUB: Subtract
   - AND: AND
   - OR: OR
   - XOR: XOR
   - SLT: Set if Less Than
   - SLTU: Set if Less Than Unsigned
   - SLL: Shift Left Logical
   - SRL: Shift Right Logical
   - SRA: Shift Right Arithmetic

3. Load/Store Instructions
   - LB/LBU: Load Byte / Load Byte Unsigned
   - LH/LHU: Load Halfword / Load Halfword Unsigned
   - LW: Load Word
   - SB: Store Byte
   - SH: Store Halfword
   - SW: Store Word

4. Control Transfer Instructions
   - BEQ: Branch if Equal
   - BNE: Branch if Not Equal
   - BLT: Branch if Less Than
   - BGE: Branch if Greater or Equal
   - BLTU: Branch if Less Than Unsigned
   - BGEU: Branch if Greater or Equal Unsigned
   - JAL: Jump and Link
   - JALR: Jump and Link Register

### Memory
- Data memory: 256 bytes (8 bits each)
- Addressable locations: 0-255

### Control Signals
- ALUOp: 4 bits (determine ALU operation)
- RegWrite: 1 bit (enable register write)
- MemWrite: 1 bit (enable memory write)
- MemRead: 1 bit (enable memory read)
- Branch: 1 bit (enable branch)
- Jump: 1 bit (enable jump)
- ALUSrc: 1 bit (select ALU source)
- PCSource: 2 bits (select PC source)

### ALU Operations
- 0000: Addition
- 0001: Subtraction
- 0010: AND
- 0011: OR
- 0100: XOR
- 0101: Shift Left
- 0110: Shift Right Logical
- 0111: Shift Right Arithmetic
- 1000: Set Less Than
- 1001: Set Less Than Unsigned

### Backward Compatibility
- Must be able to execute 2-bit RISC-V programs
- 2-bit operations should work within 8-bit framework
- Maintain same instruction encoding principles
- Provide mode switching if necessary

### Implementation Notes
- Use components built from NAND gates in Stage 1
- Extend 2-bit design principles to 8-bit
- Consider memory management and addressing modes
- Implement proper clocking and control logic
- Design for easy extension to higher bit widths

### Test Plan

- Test register file read/write operations
- Test memory load/store operations
- Verify control signal generation
- Test backward compatibility with 2-bit programs
- Verify proper handling of 8-bit arithmetic
- Test edge cases and overflow conditions
