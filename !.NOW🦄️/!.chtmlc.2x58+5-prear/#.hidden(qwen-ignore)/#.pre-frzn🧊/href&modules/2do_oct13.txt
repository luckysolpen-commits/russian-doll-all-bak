# TODO October 13 - CHTML Framework Updates

## Feature 1: Module Loading from CHTML Files

### Background
Currently, modules are passed as command-line arguments to the compiler:
`./chtmlc]23d©️™️]aaq.41]IPC++.+x test_enhanced.chtml module/+x/enhanced_game_ipc.+x`

### Goal
Enable modules to be specified within the .chtml file using `<module>` tags:
```
<module> anydir/enhanced_game_ipc.+x </module>
<module> anydir/mult_threaded.+x </module>
```

### Implementation Steps
1. Modify the CHTML parser in view.c to recognize `<module>` tags
2. Store module paths found in the CHTML file
3. Update the main.c to read modules from parsed tags instead of command line args
4. Handle multiple modules (fork multiple child processes with separate pipes)
5. Create a demo CHTML file that uses multiple modules
6. Test the implementation with existing modules
7. Update chtml_legend.md to document the new `<module>` tag

## Feature 2: Href Implementation

### Background
The framework currently doesn't support href attributes for links like:
```
<p>Learn more <a href="/products">about our products</a>.</p>
<a href="product.html" class="btn">Learn More</a>
```

### Goal
Implement href functionality to enable navigation between CHTML pages

### Implementation Steps
1. Modify the CHTML parser in view.c to recognize `<a>` tags with href attributes
2. Add UIElement support for link elements with href properties
3. Implement click handling for links in controller.c
4. Add navigation functionality to load new CHTML files based on href
5. Create a demo CHTML file that shows href functionality
6. Test internal page navigation
7. Update chtml_legend.md to document the new href support
