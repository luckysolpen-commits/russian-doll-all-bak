# Video Skew Issue - Analysis and Fix Suggestions

## Current Issue Description
Video playback is working correctly with continuous frame updates and proper functionality, but the video content appears diagonally skewed when displayed in the CHTML framework. This is a visual presentation issue rather than a functional problem.

## Current Setup
- Video playback system: Fully functional with FFmpeg integration
- Frame queuing: Working with timestamp-based rendering
- Texture updates: Happening correctly in the rendering loop
- Controls: Play, pause, stop all working properly
- Video dimensions: 1082x1672 (from console output)
- Display area: Defined by video element's x, y, width, height attributes

## Technical Details
The video texture is correctly updated via:
- `update_video_texture()` function in video_controller.c
- Called from `draw_element()` in 3.view_v0.05]textiles.c during video rendering
- Proper texture coordinates mapping:
  - glTexCoord2f(0.0f, 1.0f) for top-left
  - glTexCoord2f(1.0f, 1.0f) for top-right  
  - glTexCoord2f(1.0f, 0.0f) for bottom-right
  - glTexCoord2f(0.0f, 0.0f) for bottom-left

## Likely Causes of Diagonal Skew

1. **Aspect Ratio Mismatch:**
   - Video source: 1082x1672 (portrait orientation)
   - Display element: May have different aspect ratio
   - This could cause stretching/skewing if not properly handled

2. **Coordinate System Issues:**
   - CHTML framework uses its own coordinate system
   - Video frames might have different orientation than expected
   - Potential mismatch between video coordinate system and OpenGL coordinate system

3. **Texture-to-Quad Mapping:**
   - The video texture might not be mapped correctly to the display quad
   - Quad vertices might be skewed due to coordinate transformations

4. **OpenGL State Issues:**
   - Matrix transformations from other UI elements might affect video rendering
   - Transformations applied to parent containers might impact video display

## Potential Solutions

### 1. Preserving Aspect Ratio
- Calculate the video's native aspect ratio (1672/1082 ≈ 1.54)
- Scale the video to fit within the display area while maintaining aspect ratio
- Add letterboxing/pillarboxing if needed

### 2. Coordinate System Alignment
- Check if video frames need to be flipped vertically or horizontally
- Ensure texture coordinate mapping matches the actual video frame orientation
- Consider that video frames might be stored in different coordinate orientations

### 3. Safe Frame Mapping
- Create a centered quad that preserves the original aspect ratio
- Calculate proper width/height for display based on original aspect ratio
- Example pseudo-code:
  ```
  float video_aspect = (float)video_width / (float)video_height;
  float display_aspect = (float)element_width / (float)element_height;
  float render_width, render_height;
  
  if (video_aspect > display_aspect) {
      render_width = element_width;
      render_height = element_width / video_aspect;
  } else {
      render_height = element_height;
      render_width = element_height * video_aspect;
  }
  ```

### 4. Isolated Rendering
- Temporarily clear any active transforms before rendering video
- Save and restore OpenGL state around video rendering
- Ensure no inherited transformations affect video geometry

### 5. Texture Flip Check
- Video frames from FFmpeg might be upside down relative to OpenGL expectations
- Try flipping texture coordinates: swap top/bottom coordinates
- Consider the pixel format and frame orientation from decoder

## Files to Focus On
- `3.view_v0.05]textiles.c` - draw_element function, video rendering section
- `video_controller.c` - update_video_texture function, especially texture coordinate handling
- Consider texture format issues in color space conversion
