# Remaining Media Support Plan

## Overview
This document outlines the goals and implementation plan for adding audio and video support to the CHTML framework, building on the successful texture implementation for module shapes.

## Current State
- ✅ UI element images work with `<img src="...">` 
- ✅ Module shapes can be textured with `texture_src=` parameter in SHAPE commands
- ✅ Non-blocking texture loading system preserves GUI responsiveness
- ✅ Working with both 2D and 3D rendering modes

## Audio Support Goals

### 1. Audio Element Implementation
- Add `<audio src="file.mp3">` tag support to CHTML
- Implement `load_audio_from_file()` function to preload audio files
- Support basic audio controls: play, pause, volume, loop

### 2. Module Audio Commands
- Extend IPC protocol with AUDIO commands similar to SHAPE
- Example: `AUDIO;play;sound_effect.mp3;volume=0.8;loop=false`
- Example: `AUDIO;stop;music_track.mp3`

### 3. Technical Implementation
- Use PulseAudio for cross-platform audio output (already available in existing code)
- Implement audio channel management for multiple simultaneous sounds
- Add audio preloading similar to texture preloading
- Ensure non-blocking audio loading like the texture system

### 4. Integration Points
- Add to UIElement structure: audio-specific fields
- Update parsing functions to handle AUDIO commands
- Add rendering indicators for audio playback (visual feedback)
- Integrate with event system for audio triggers

## Video Support Goals

### 1. Video Element Implementation  
- Add `<video src="file.mp4">` tag support to CHTML
- Support basic video controls: play, pause, seek, volume

### 2. Module Video Commands
- Extend IPC protocol with VIDEO commands
- Example: `VIDEO;play;animation.mp4;x=100;y=100;width=320;height=240`
- Support video as textures for 3D shapes

### 3. Technical Implementation
- Use FFmpeg for decoding support (libavcodec, libavformat, libswscale)
- Render video frames to OpenGL textures for display
- Synchronize audio and video streams using PulseAudio
- Implement frame queuing for smooth playback

### 4. Integration Points
- Add to UIElement structure: video-specific fields
- Implement video texture pipeline (frames → OpenGL textures)
- Add video timeline/seek controls
- Support embedding videos in canvases

## Implementation Strategy

### Phase 1: Audio Foundation
1. Create proof-of-concept audio player using PulseAudio
2. Integrate audio loading into the model-view-controller architecture
3. Add basic AUDIO commands to IPC protocol
4. Test with simple audio events


### Phase 2: Video Foundation  
5. Create video player using FFmpeg + OpenGL
6. Implement video texture system (similar to image textures)
7. Add VIDEO commands to IPC protocol
8. Test with simple video playback

### Phase 3: CHTML Integration
9. Add `<audio>` and `<video>` tag parsers
10. Implement DOM elements for audio/video
11. Add event handlers for media controls
12. Full integration testing

## Architecture Considerations

### Memory Management
- Audio/video buffering without causing memory issues
- Frame caching strategies for smooth playback
- Resource cleanup for changing media

### Performance
- Non-blocking loading for all media (like texture system)
- Efficient texture updates for video frames
- Low-latency audio playback
- Thread safety for async operations

### GUI Responsiveness
- All media loading must be non-blocking
- Separate threads for decoding where needed
- Progress indicators for long operations

## Dependencies
- FFmpeg libraries: libavcodec, libavformat, libavutil, libswresample
- PulseAudio: libpulse-simple, libpulse
- OpenGL texture extensions for video frames

The foundation is already laid with the texture system, so implementing audio and video should follow similar patterns while leveraging the existing architecture.
