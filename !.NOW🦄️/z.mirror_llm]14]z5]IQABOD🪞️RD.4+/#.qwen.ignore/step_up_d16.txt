This document outlines the technical steps to transition your homemade LLM from a heuristic-based training model to a standard deep learning pipeline using backpropagation and gradient descent.

## Introduction: The Goal

Our objective is to replace the current training mechanism, which relies on external modules modifying a text file, with an industry-standard, mathematically-grounded training loop. This involves two core concepts:

1.  **Loss Calculation**: Accurately measuring how "wrong" the model's predictions are.
2.  **Backpropagation & Gradient Descent**: Systematically calculating the error contribution of each weight and updating it to reduce the error.

This process will make your model more powerful, stable, and comparable to modern LLM architectures. It requires significant changes but provides a clear path to a more robust system.

---

## Step 1: Real Loss Calculation

Currently, your `loss.txt` is populated by simulated values. We need to replace this with a real, meaningful score that guides the learning process. For language models, the standard is **Cross-Entropy Loss**.

### Conceptual Overview

1.  **Model Prediction**: For a given input token, the model's forward pass produces a vector of raw scores called `logits`. The size of this vector is equal to your vocabulary size.
2.  **Convert to Probabilities**: The `logits` are converted into a probability distribution using the **Softmax function**. This ensures all values are between 0 and 1, and they all sum to 1.0. The softmax value for a specific token `i` is calculated as: `exp(logit_i) / sum(exp(all_logits))`.
3.  **Measure the Error**: The goal is to maximize the probability assigned to the correct next token (the `target`). Cross-Entropy Loss measures this. If the model assigns a probability of `0.9` to the correct target, the loss is low. If it assigns `0.01`, the loss is high. The formula is simply: **`Loss = -log(probability_of_the_correct_target_token)`**.

### Implementation Plan

1.  **Decommission Fake Loss Functions**: Delete the `calculate_sequence_loss...` functions from `generation_module.c`. They will no longer be needed.

2.  **Create a New `calculate_loss` Function**: Add this function to `generation_module.c`. It will be the heart of your scoring system.

    ```c
    // In generation_module.c

    /**
     * Calculates the cross-entropy loss.
     *
     * @param probabilities The output from the softmax function (must sum to 1.0).
     * @param target_token_id The index of the correct next token.
     * @param vocab_size The total size of the vocabulary.
     *
     * @return The cross-entropy loss value.
     */
    float calculate_cross_entropy_loss(float* probabilities, int target_token_id) {
        // Ensure the probability is not zero to avoid log(0) which is -infinity.
        // A small epsilon can be added for numerical stability if needed, but a good
        // softmax implementation should prevent this.
        float prob_of_target = probabilities[target_token_id];
        if (prob_of_target == 0.0f) {
            return -logf(1e-9); // Return a very large loss value
        }
        return -logf(prob_of_target);
    }
    ```

3.  **Modify the `train` Mode Logic**: The main loop inside the `train` mode of `generation_module.c` needs a complete overhaul. The `system()` calls should be removed and replaced with a proper training step.

    ```c
    // Revised logic for the "train" mode in generation_module.c main()

    // ... inside the training loop (iterating through epochs and corpus tokens) ...

    for (int i = 0; i < num_corpus_tokens - 1; i++) {
        int current_token_id = corpus_token_ids[i];
        int target_token_id = corpus_token_ids[i+1];

        // 1. FORWARD PASS: Run the model to get logits
        // The `pos` argument is the position in the sequence
        float* logits = transformer_forward(model, vocab, &state, current_token_id, i);

        // 2. SOFTMAX: Convert logits to probabilities
        // The softmax function should modify the logits array in-place to save memory.
        softmax(logits, model->vocab_size); // You already have a softmax function

        // 3. LOSS CALCULATION: Calculate the real loss
        float loss = calculate_cross_entropy_loss(logits, target_token_id);

        // --- BACKPROPAGATION WILL GO HERE (See Step 2) ---

        // 4. LOGGING: Log the real loss
        printf("Epoch %d, Step %d, Loss: %f\n", epoch, i, loss);
        // Write to loss.txt as before
        FILE *loss_file = fopen("loss.txt", "a");
        if (loss_file) {
            fprintf(loss_file, "%d,training,step,%d,%.6f\n", epoch, i, loss);
            fclose(loss_file);
        }
    ```

After this step, your `train` command will run a full forward pass for each token in your training data and compute a mathematically correct loss value that truly represents your model's performance. This loss value is the critical signal needed for learning.

## Step 2: Backpropagation and Gradient Descent

This is the most complex but most important step. It's how the model *learns* from the loss score. **Backpropagation** calculates the error contribution of every weight, and **Gradient Descent** updates those weights to reduce the error.

### CRITICAL: Mandatory Architectural Shift

Before implementing backpropagation, you **must** transition your model from its current token-centric design to a standard **layer-centric architecture**.
Your current `transformer_forward` function uses per-token `q_proj`, `k_proj`, and `v_proj` vectors and performs an element-wise multiplication. Backpropagating through this is non-standard and will not behave like a transformer.

**Action Required:**
1.  **Modify `transformer_forward`**: You must replace the custom QKV calculation with the standard `matmul` (matrix multiplication) approach, as seen in `run]karp=999.loc.c`. Your `SimpleModel` struct already has the correct weight matrices (`wq`, `wk`, `wv`). The code should look like this:
    ```c
    // REPLACE THIS:
    // for(int d = 0; d < dim; d++) {
    //     state->q[d] = state->xb[d] * token_q_proj[d % dim];
    // }
    // ...etc for k, v

    // WITH THIS (from run]karp=999.loc.c):
    matmul(state->q, state->xb, model->wq + l*dim*dim, dim, dim);
    matmul(state->k, state->xb, model->wk + l*dim*kv_dim, dim, kv_dim);
    matmul(state->v, state->xb, model->wv + l*dim*kv_dim, dim, kv_dim);
    ```
2.  **Decommission Heuristic Modules**: The external executables (`feedback_tx_module`, `attention_module`, etc.) and the `system()` calls to them must be completely removed from your training loop. Their function will be fully replaced by this new backpropagation pipeline.

### Implementation Plan

#### 1. Add Gradient Buffers

For every weight parameter in your model, you need a corresponding buffer to store its gradient. Modify the `SimpleModel` struct:

```c
typedef struct {
    // --- Existing weights ---
    float *token_embedding_table; // (vocab_size, dim)
    float *rms_att_weight; // (layer, dim)
    float *wq; // (layer, dim, dim)
    // ... all other weights ...
    float *wcls;

    // --- NEW: Gradients for all weights ---
    float *grad_token_embedding_table;
    float *grad_rms_att_weight;
    float *grad_wq;
    // ... one gradient buffer for each weight pointer ...
    float *grad_wcls;

    // ... other model parameters ...
} SimpleModel;

// You also need gradient buffers for the run state activations
typedef struct {
    // --- Existing activations ---
    float *x; // (dim,)
    float *xb; // (dim,)
    // ... etc ...
    float *logits; // (vocab_size,)

    // --- NEW: Gradients for all activations ---
    float *grad_x;
    float *grad_xb;
    // ... etc ...
    float *grad_logits;

```
Remember to `malloc` and `free` these new buffers, and to zero them out at the start of each backpropagation pass.

#### 2. Create `_backward` Functions

For every function in the forward pass, you must implement its derivative (the backward pass). This involves applying the chain rule of calculus. This is a complex task. Here is the conceptual structure:

*   **`softmax_backward(grad_logits, probabilities, target_token_id)`**: The backward pass for softmax and cross-entropy loss together is simple. The initial gradient in `grad_logits` is just the `probabilities` array, with `1.0` subtracted from the probability of the `target_token`.
    ```c
    // grad_logits is pre-filled with the probabilities from softmax
    grad_logits[target_token_id] -= 1.0f;
    ```
*   **`matmul_backward(grad_xout, grad_x, grad_w, x, w, n, d)`**: This computes the gradients for the input (`grad_x`) and the weights (`grad_w`) given the gradient from the output (`grad_xout`).
*   **`rmsnorm_backward(...)`**: Calculates gradients for the RMSNorm input and weight.
*   And so on for every operation (`swiglu_backward`, `rope_backward`, `attention_backward`). The attention backward pass is the most involved.

#### 3. Implement `transformer_backward()`

This function orchestrates the entire backward pass by calling the individual `_backward` functions in the reverse order of the forward pass.

After backpropagation has calculated the gradients for all weights, this function applies the update rule.

```c
// In generation_module.c

void update_weights(SimpleModel* model, float learning_rate) {
    // Iterate over every single parameter in the model
    int num_params = /* total number of weights */;
    float* all_weights[] = { model->wq, model->wk, /* ... all other weights ... */ };
    float* all_grads[] = { model->grad_wq, model->grad_wk, /* ... all other grads ... */ };

    for (int p = 0; p < num_params; p++) {
        // The core gradient descent update rule
```
