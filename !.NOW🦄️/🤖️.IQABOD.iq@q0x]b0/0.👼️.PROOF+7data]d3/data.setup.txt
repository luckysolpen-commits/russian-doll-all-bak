✦ Let me clarify this concept. In transformer models like yours:

   1. Word embeddings are unique per word: Each word in the vocabulary has its
      own unique embedding vector (which is 16-dimensional in your case).

   2. Transformation matrices (weights) are shared: The attention weight matrices
      and feed-forward network weights are shared across all positions in the
      sequence. These are model parameters that are learned during training and
      apply to any input they receive.

  For example, if you have a weight matrix for the feed-forward layer (say
  16×32), it transforms any 16-dimensional input vector to a 32-dimensional
  output vector, regardless of which word it came from.

  Positional encoding (like ROPE) is added to the word embeddings to provide
  position information, but the transformation weights themselves are not
  unique per word.

  The model doesn't store separate weights for each individual word. Instead,
  it has one set of weights that applies to all words. This is more efficient
  and allows the model to generalize to unseen word combinations.

  So when you want to export data, you have:
   - A unique embedding vector for each word (16 values per word)
   - Shared weight matrices for transformations (same for all words)
   - Shared bias vectors (same for all words)

  Would you like the export to duplicate the shared weights for each word, or
  would you prefer a different format that acknowledges that some parameters
  are shared across all words?
