### 2. Key Discoveries and Learning Points

#### About the Training Process:
- The trainer executable is: `./+x/trainer.+x`
- It needs a path to a vocab model file as an argument (not a curriculum bank text file)
- Usage: `./+x/trainer.+x ./curriculum/guess_the_letter/guess_the_letter.txt`
- The vocab model file format: "number word embedding pe weight bias1 bias2 bias3 bias4"
- Training creates output files in the same directory as the vocab model with names like:
  - attention_model.txt, mlp_model.txt, output_layer.txt
  - grad files and various intermediate matrices

#### About the Chat System:
- Use `./+x/chatbot_moe_v1.+x meta_curriculum_bank.txt "prompt" length temperature`
- Or use the script `sh run_FAME_chat.sh "prompt" length temperature`
- The meta_curriculum_bank.txt should contain the path to the vocab model
- Example: `./+x/chatbot_moe_v1.+x meta_curriculum_bank.txt "f a r" 1 0.1`

#### Architecture Notes:
- Main components: vocab_model.c, chatbot_moe_v1.c, trainer.c, forward_prop.c, backward_prop.c, optimizer.c
- The attention mechanism uses Q, K, V matrices with context vectors
- The model processes sequences by aggregating context from all tokens (not just last token)
- Multi-layer perception (MLP) follows the attention mechanism

### 6. Common Issues Discovered
- Previously confused curriculum bank file with vocab model file (they serve different purposes)
- Trainer needs direct vocab model path, not curriculum list
- Loss values staying constant may indicate convergence or insufficient learning
- Gradient norms should stay bounded (implemented clipping at 1.0 norm)
- Need to ensure vocab model format matches expected "number word embedding pe weight bias1 bias2 bias3 bias4"
