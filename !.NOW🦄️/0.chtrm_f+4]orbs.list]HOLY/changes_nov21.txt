# Changes Made for Emoji Palette Feature (November 21, 2025)

## Summary
Added a 1x8 focusable canvas for an ASCII symbol palette to the CHTM application with support for keyboard navigation and selection.

## Key Changes Made:

### 1. Updated index_with_canvas.chtm file:
- Added a new canvas element with ID "emoji_palette" and dimensions 8x1
- Set key="3" to make it a focusable element
- Added appropriate label and color for visual distinction

### 2. Enhanced canvas_player_v.01.c module:
- Added palette-specific data structures:
  - `palette_symbols` array with 8 ASCII symbols: {' ', '!', '#', '$', '%', '^', '&', '*'}
  - `selected_palette_index` to track current selection
  - `palette_canvas` for rendering the palette
- Implemented new functions:
  - `init_palette()` to initialize the palette
  - `display_palette()` to render the palette with selected item highlighted with brackets
  - `move_palette_selection()` to handle navigation (left/right arrows and numeric keys 0-7)
- Updated main command loop to handle new commands:
  - `DISPLAY_PALETTE` command for rendering the palette specifically
  - `PALETTE_SIZE:` command for palette-specific initialization
  - `PALETTE_MOVE:` command for palette navigation/selection

### 3. Modified chtm_c5.c renderer:
- Updated canvas rendering logic to distinguish between game canvas and palette canvas
- Added conditional logic to send different commands based on canvas ID:
  - `DISPLAY` command for game canvas (existing)
  - `DISPLAY_PALETTE` command for palette canvas (new)
- Implemented different response handling for palette vs game canvas
- Enhanced arrow key handling to process palette navigation when palette canvas is focused:
  - Right arrow: sends PALETTE_MOVE:R for palette canvas
  - Left arrow: sends PALETTE_MOVE:L for palette canvas
- Added numeric key support (0-7) for direct palette item selection when focused on palette
- Modified canvas reading logic to handle different line counts for different canvas types

## How the System Distinguishes Between Canvases:

The system differentiates between the game canvas and the palette canvas using the `id` attribute of each canvas element:

1. **In the CHTM file**, each canvas has a unique ID:
   - Game canvas has `id="game_canvas"`
   - Palette canvas has `id="emoji_palette"`

2. **In the renderer code**, conditional checks look at `strcmp(el->id, "emoji_palette") == 0` to determine if the current canvas being processed is the emoji palette.

3. **Based on this ID check**, the renderer:
   - Sends different commands: `DISPLAY_PALETTE` for the palette canvas vs `DISPLAY` for the game canvas
   - Reads different numbers of lines: 1 line for the palette vs 8 lines for the game canvas
   - Handles input differently: arrow keys move the selection in the palette vs the player in the game canvas

This ID-based approach allows multiple canvases to coexist in the same application while being processed with different logic. This is a common pattern in UI frameworks where element IDs serve as unique identifiers that allow different behavior based on the specific component being interacted with.

## Why ASCII Emojis have different spacing than dots in the game canvas:

The difference in spacing comes from how each canvas displays its content:

1. **Game Canvas** (8x8): Uses the format "char ", where each character is followed by a space, creating the pattern ". . . . . . . ."

2. **Emoji Palette** (8x1): The module formats it as "[ ]" or " [char] " depending on selection state, which has different spacing. The palette renders as something like "[ ]  !   #   $   %   ^   &   * " where selected items show brackets and spacing is different.

The palette items have different spacing because:
- The module sends the formatted palette string directly (with different spacing logic)
- Each palette item is formatted as " [char] " when unselected or "[char]" when selected
- The palette display function was designed with different spacing logic than the game canvas

## New Feature Capabilities:
- Arrow keys (left/right) when palette is focused to navigate between symbols
- Numeric keys (0-7) when palette is focused to directly select a symbol
- Visual highlighting of the currently selected symbol with brackets
- Focus navigation between all interactive elements including the palette
- The palette updates in real-time as selections are made
