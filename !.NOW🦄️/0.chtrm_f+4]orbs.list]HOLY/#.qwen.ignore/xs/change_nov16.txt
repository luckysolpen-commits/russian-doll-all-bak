# Changelog - November 16, 2025

This document details the foundational refactoring of the C-MVC application to support robust UI parsing and navigation, aligning with the CHTM specification.

## 1. Core Architectural Changes

The primary change was the replacement of the simple string-replacement rendering system with a professional, two-stage **"Parse then Render"** architecture.

-   **Parser (`view.c`):** The application no longer relies on fragile string searching. When a CHTM file is loaded, it is now fully parsed into a structured, in-memory tree of `UIElement` objects. This tree accurately represents the UI's components and their relationships.
-   **Renderer (`view.c`):** The rendering logic now iterates through the pre-parsed `UIElement` tree to draw the UI. This decouples rendering from parsing, making the system more robust, predictable, and easier to extend.

## 2. New Features & CHTM Specification

The new parser and controller enable several key features and align the CLI with the broader CHTM specification.

### Supported CHTM Tags:

-   **`<link href="..." key="..." label="...">`**: Creates a navigable link. The `key` attribute is used for selection in the CLI.
-   **`<button onClick="..." key="..." label="...">`**: Creates a clickable button. The `onClick` attribute contains a command string that the controller can process.
-   **`<text label="..." />`**: Displays a simple line of text.
-   **`<panel>`**: A structural tag used to group elements. It is parsed correctly to establish parent-child relationships, though it has no visual output in the CLI yet.
-   **`<module>`**: The path to a module executable should now be placed between the tags, like this: `<module>path/to/module.elf</module>`. The old `src="..."` attribute is deprecated.

### How to Use the New Features:

-   **Navigation:** To use a `<link>`, press the corresponding `key` number, followed by the `Enter` key. The application will load the CHTM file specified in the `href` attribute.
-   **Buttons:** To use a `<button>`, press the corresponding `key` number, followed by `Enter`. The controller will process the `onClick` command string. Currently, it prints a status message confirming the action (e.g., "Button clicked: YOUR_COMMAND").

## 3. Codebase Changes

-   **`view.c`**:
    -   The `UIElement` struct was significantly expanded to hold more data (type, onClick, parent_index, etc.).
    -   The `print_ascii_grid` function was completely rewritten to house the new parser and renderer.
    -   The old, naive parsing logic and helper functions were removed.
-   **`controller.c`**:
    -   The `UIElement` struct was updated to match `view.c`.
    -   The `process_input` function was refactored to be action-aware. It now inspects the `type` of a selected `UIElement` to decide whether to navigate (`link`) or execute a command (`button`).
-   **`0.main.c`**:
    -   The default startup file is now `chtm/index.chtm`.
    -   The `parse_chtm_for_module` function was updated to support the new `<module>...</module>` tag syntax and is no longer a fatal error if a page has no module.
-   **`chtm/` directory**:
    -   Added `index.chtm` and `page2.chtm` as a working demonstration of the new link and button features.

## 4. Compiling the Application

To compile the project, use the provided shell script. It correctly compiles all `.c` files and links them into a single executable.

```bash
bash xh.compile.dir.link.+x.sh
```

This will produce an executable named after the project directory (e.g., `simple-mvc]e3=chtm&ipc]+19.+x`).
