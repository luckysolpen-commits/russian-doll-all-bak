# Hangman Game Implementation Plan - December 10, 2025

## Overview
Create a terminal-based Hangman game using the CHTM framework with the following components:
- CHTM file for UI layout and display
- C module for game logic and state management
- Vocab file support for word selection

## Components to Create

### 1. CHTM File: `chtm/hangman.chtm`
- Game title display
- Hangman visual representation (canvas)
- Display for current word with blanks and guessed letters
- Display for incorrect guesses
- Input field for letter guesses
- Game status display (win/lose messages)
- Controls (restart, quit)

### 2. C Module: `modules/hangman/hangman.c`
- Game state management (current word, guessed letters, incorrect guesses)
- Word selection from vocabulary file
- Guess processing and validation
- Game win/lose logic
- Hangman drawing logic
- File I/O for vocab loading

## Game Logic
- Random word selection from vocab file
- Track correctly and incorrectly guessed letters
- Maintain game state (attempts remaining: 6 total)
- Win condition: word completed before hangman finished
- Lose condition: hangman completed (6 incorrect guesses)

## Vocabulary File Support
- Flexible vocab file loading (default: `hangman_chat]vocab.txt`)
- Line-by-line word loading
- Random word selection from file

## UI Elements
1. Title: "Hangman Game"
2. Hangman canvas: Visual representation of hangman progress
3. Word display: Shows current state of word (e.g., "_ _ L _ _")
4. Incorrect guesses: Shows letters that were guessed but not in word
5. Input field: For entering letter guesses
6. Status: Win/lose messages
7. Controls: Restart game, quit options

## State Variables to Track
- `current_word`: The word to guess
- `display_word`: The word with blanks and revealed letters
- `guessed_letters`: All letters that have been guessed
- `incorrect_letters`: Letters guessed that were wrong
- `incorrect_count`: Number of incorrect guesses (0-6)
- `game_status`: Current state (playing, won, lost)
- `user_input`: Current guess from CLI input
- `message`: Status messages for player

## Hangman Stages
The hangman will be drawn progressively:
1. No body parts (0 incorrect)
2. Head only (1 incorrect)
3. Head + body (2 incorrect)
4. Head + body + 1 arm (3 incorrect)
5. Head + body + 2 arms (4 incorrect)
6. Head + body + 2 arms + 1 leg (5 incorrect)
7. Complete hangman (6 incorrect - game over)

## File Structure
- `chtm/hangman.chtm`: The UI layout file
- `modules/hangman/hangman.c`: The game logic module
- `modules/hangman/hangman_data/hangman_chat]vocab.txt`: Vocabulary file
- `modules/+x/hangman.+x`: Compiled module executable

## Implementation Steps
1. Create vocabulary file (already exists)
2. Write hangman.c module with game logic
3. Create hangman.chtm UI file
4. Compile the module
5. Test the game
