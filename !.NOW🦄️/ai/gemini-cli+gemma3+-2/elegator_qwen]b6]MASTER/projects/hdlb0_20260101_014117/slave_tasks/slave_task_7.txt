# Slave Task 7: Fix HDLb0 Addressing for Emulator Compatibility

## Objective
Update the RISC-V to HDLb0 assembler to generate proper memory addresses that are compatible with the HDLb0 emulator.

## Problem Identified
Current HDLb0 files generated by the assembler use addresses below 16 (like 4, 8) which are outside the valid RAM range. The HDLb0 emulator reserves addresses 0-15 for special purposes, with general-purpose RAM starting at address 16.

## Specific Tasks
1. Review the current riscv_to_hdlb0_assembler.py to identify where memory addresses are generated
2. Update the assembler to only use addresses 16 and above for general RAM operations
3. Ensure all generated HDLb0 files use valid addresses in the range 16-255
4. Test the updated assembler with existing RISC-V programs
5. Regenerate all HDLb0 output files with correct addressing
6. Verify the generated files work with the emulator

## Deliverables
- Updated riscv_to_hdlb0_assembler.py with proper addressing
- Corrected HDLb0 files in ./riscv_2bit/hdlb0_output/ 
- Test results showing emulator compatibility
- Updated documentation explaining the addressing scheme

## Implementation Notes
- RAM addresses should start from 16 (0x10) as per HDLb0 specification
- Addresses 0-15 are reserved for special purposes
- Maintain the same functionality while fixing addressing
- Ensure backward compatibility with existing designs
- Test with the emulator to verify functionality

## Output Location
- Updated assembler: ./riscv_to_hdlb0_assembler.py
- Corrected HDLb0 files: ./riscv_2bit/hdlb0_output/
- Test results: ./test_results/emulator_tests/

## Verification
- All generated HDLb0 files should use addresses >= 16
- Files should execute successfully on the HDLb0 emulator
- Functionality should remain the same as before
- All existing tests should still pass
