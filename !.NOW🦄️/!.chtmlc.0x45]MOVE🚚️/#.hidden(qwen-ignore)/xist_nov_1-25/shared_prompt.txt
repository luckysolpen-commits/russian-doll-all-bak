# Shared Memory Module System - Next Iteration Prompt

## Current State Analysis
- Successfully implemented shared memory infrastructure in the chtml model (2.model_v0.01.c)
- Added support for protocol="shared" in CHTML module tags
- Created foundation with ModuleSharedMem structure, semaphores, and communication functions
- Maintained full backward compatibility with pipe-based modules
- System can now handle both pipe-based and shared memory protocols simultaneously

## Core Issue Identified
The current implementation creates shared memory infrastructure in the model, but external module executables (like shared_player_movement.+x) are still running as separate forked processes using stdin/stdout pipes rather than actually using the shared memory infrastructure that was created for them.

## Desired Architecture
Need to modify the module execution system so that:
1. When protocol="shared" is specified, the module directly uses the shared memory infrastructure instead of stdin/stdout pipes
2. Modules execute within the same process space or have direct access to the created shared memory segments
3. Eliminate the pipe-based communication for shared memory modules, using only shared memory + semaphores

## Implementation Requirements
1. **Change module execution flow**: Instead of fork/exec with pipes, modules with protocol="shared" should use the shared memory segments created by the model
2. **Module-Infrastructure Integration**: Either integrate module logic directly into the main process or ensure the module executable can properly connect to pre-existing shared memory segments
3. **Timing Coordination**: Ensure shared memory segments exist before the module tries to access them
4. **Communication Channel Replacement**: For shared protocol, replace stdin/stdout pipe communication with shared memory + semaphore communication

## Success Criteria
- When using `<module protocol="shared">`, the module communicates exclusively via shared memory segments (not pipes)
- Significant performance improvement over pipe-based communication for shared memory modules
- Maintain backward compatibility for protocol="pipe" modules
- Demonstrate the 3D/2D player movement demo using true shared memory communication

## Next Steps
Modify the module initialization and communication system in 2.model_v0.01.c and related files to ensure that when protocol="shared" is specified, the module uses the shared memory infrastructure instead of stdin/stdout pipe communication.
