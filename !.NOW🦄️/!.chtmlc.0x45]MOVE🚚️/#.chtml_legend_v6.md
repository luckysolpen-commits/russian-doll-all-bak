# C-HTML Legend v6

This document describes the available elements and attributes in the C-HTML framework.

## Elements

### `<window>`

The root element of a C-HTML document.

**Attributes:**

*   `title` (string): The title of the application window.
*   `x` (int): The x-coordinate of the window on the screen (default: 100).
*   `y` (int): The y-coordinate of the window on the screen (default: 100).
*   `width` (int): The width of the window in pixels (default: 800).
*   `height` (int): The height of the window in pixels (default: 600).
*   `color` (hex): The background color of the window (e.g., "#222222").
*   `label` (string): An optional label for the window (also used as window title).
*   `alpha` (float): The transparency level of the window (0.0 = fully transparent, 1.0 = fully opaque, default: 1.0).
*   `z_level` (int): The rendering priority/depth level (0-99, default: 0, higher values render on top).

### `<header>`

A container for header content.

**Attributes:**

*   `x` (int): The x-coordinate of the header (default: 0).
*   `y` (int): The y-coordinate of the header.
*   `width` (int): The width of the header.
*   `height` (int): The height of the header.
*   `color` (hex): The background color of the header (e.g., "#333333").
*   `alpha` (float): The transparency level of the header (0.0 = fully transparent, 1.0 = fully opaque, default: 1.0).
*   `z_level` (int): The rendering priority/depth level (0-99, default: 0, higher values render on top).
*   `label` (string): An optional label for the header.

### `<text>`

A static text element.

**Attributes:**

*   `id` (string): A unique identifier for the text element.
*   `value` (string): The text to display.
*   `x` (int): The x-coordinate of the text.
*   `y` (int): The y-coordinate of the text.
*   `color` (hex): The color of the text (e.g., "#FFFFFF").
*   `label` (string): An alternative way to specify the text content (takes precedence over value).
*   `alpha` (float): The transparency level of the text (0.0 = fully transparent, 1.0 = fully opaque, default: 1.0).
*   `z_level` (int): The rendering priority/depth level (0-99, default: 0, higher values render on top).

### `<slider>`

A slider for selecting a value from a range. Supports both horizontal (width > height) and vertical (height > width) orientations automatically.

**Attributes:**

*   `id` (string): A unique identifier for the slider.
*   `x` (int): The x-coordinate of the slider.
*   `y` (int): The y-coordinate of the slider.
*   `width` (int): The width of the slider track.
*   `height` (int): The height of the slider track.
*   `min` (int): The minimum value of the slider.
*   `max` (int): The maximum value of the slider.
*   `value` (int): The current value of the slider.
*   `step` (int): The increment/decrement step for the slider value.
*   `color` (hex): The background color of the slider track (e.g., "#333333").
*   `alpha` (float): The transparency level of the slider (0.0 = fully transparent, 1.0 = fully opaque, default: 1.0).
*   `z_level` (int): The rendering priority/depth level (0-99, default: 0, higher values render on top).
*   `label` (string): An optional label for the slider.

**Behavior:**
*   Orientation is automatically determined by aspect ratio (height > width = vertical slider)
*   Sends `SLIDER:<id>:<value>` to external modules when moved
*   Supports both mouse click and drag interaction

### `onClick` handlers

Specialized event handlers for various UI interactions.

**Legacy handlers:**

*   `switch_to_2d_handler` (string): Switches canvas elements to 2D rendering mode.
*   `switch_to_3d_handler` (string): Switches canvas elements to 3D rendering mode.
*   `button_increment_handler` (string): Increments counter value (legacy).
*   `button_reset_handler` (string): Resets counter value (legacy).
*   `button_click_handler` (string): Basic button click (legacy).
*   `canvas_click_handler` (string): Canvas click handler (legacy).
*   `menu_click_handler` (string): Menu click handler (legacy).

**Dynamic Event System:**

Modern CHTML applications now use a dynamic event system that supports arbitrary handler names. Instead of being limited to hardcoded handlers, any event name can be used with the format: `EVENT:<handler_name>` where `<handler_name>` is passed to the module for processing.

**Custom handler patterns:**
*   Handlers containing "increment" or "Increment" trigger counter increment in compatible modules
*   Handlers containing "reset" or "Reset" trigger counter reset in compatible modules
*   Handlers containing "click" or "Click" are processed as click events
*   Any other custom handler names are passed to the module as-is

**Backward compatibility:**
*   Legacy handler names continue to work as before
*   New modules can handle both legacy commands and dynamic events

### `<canvas>`

A drawable area for custom graphics rendering and interaction with external modules.

**Attributes:**

*   `id` (string): A unique identifier for the canvas.
*   `x` (int): The x-coordinate of the canvas.
*   `y` (int): The y-coordinate of the canvas.
*   `width` (int): The width of the canvas.
*   `height` (int): The height of the canvas.
*   `label` (string): An optional label for the canvas.
*   `view_mode` (string): The rendering mode for the canvas - "2d" for 2D rendering or "3d" for 3D rendering (default: "2d").
*   `fov` (float): Field of view for 3D rendering (default: 45.0, affects perspective)
*   `onClick` (string): The name of the C function to call when the canvas is clicked.
*   `color` (hex): The background color of the canvas (e.g., "#333333").
*   `alpha` (float): The transparency level of the canvas (0.0 = fully transparent, 1.0 = fully opaque, default: 1.0).
*   `z_level` (int): The rendering priority/depth level (0-99, default: 0, higher values render on top).

**Behavior:**
*   Can receive click events and sends `CANVAS:<x>,<y>` to external modules (coordinates relative to canvas origin)
*   Supports custom shape rendering from external modules using the communication protocol
*   Updates dynamically based on messages from connected modules
*   Supports 3D camera positioning and perspective transformation

### `<module>`

A special element that specifies an external module to load and run with the page.

**Attributes:**

*   `protocol` (string): Communication protocol - "pipe" (default) or "shared" for shared memory

**Content:**

*   The file path to the executable module (e.g., "module/+x/page_test.+x" or "module/+x/color_grid.+x")

**Behavior:**
*   Loads and runs the specified external module process
*   Establishes two-way communication via stdin/stdout pipes (or shared memory if protocol="shared")
*   Allows the module to send updates to UI elements using the communication protocol
*   Modules can be used to provide dynamic functionality like counters, graphics, or data processing

*   Can be used with other onClick handlers simultaneously

## Module Communication Protocol

The C-HTML framework supports communication with external modules via standard input/output pipes or shared memory. This enables dynamic UI updates and custom rendering in canvas elements.

### Messages from UI to Module:

*   `SLIDER:<id>:<value>` - Sent when a slider value changes
*   `CANVAS:<x>,<y>` - Sent when a canvas is clicked (coordinates relative to canvas origin)
*   `EVENT:<handler_name>` - Sent when any onClick handler is triggered (dynamic event system)
*   `UP`, `DOWN`, `LEFT`, `RIGHT` - Arrow key presses sent to modules
*   `INCREMENT` - Sent for legacy increment operations (backward compatibility)
*   `RESET` - Sent for legacy reset operations (backward compatibility)

### Messages from Module to UI:

*   `CLEAR_SHAPES` - Clear all previously rendered shapes
*   `SHAPE;type;label;x;y;z;width;height;depth;r;g;b;a` - Render a shape in the canvas (OLD FORMAT - 12 fields)
    *   `type`: Shape type (e.g., "SQUARE", "RECT", "CIRCLE", "TRIANGLE")
    *   `label`: Unique identifier for the shape
    *   `x,y,z`: 3D coordinates
    *   `width,height,depth`: Dimensions
    *   `r,g,b,a`: RGBA color values (0.0-1.0)
*   `SHAPE;type;label;x;y;z;width;height;depth;r;g;b;a;rot_x;rot_y;rot_z;scale_x;scale_y;scale_z` - Render a shape in the canvas (NEW FORMAT - 18 fields)
    *   `type`: Shape type (e.g., "SQUARE", "RECT", "CIRCLE", "TRIANGLE")
    *   `label`: Unique identifier for the shape
    *   `x,y,z`: 3D coordinates
    *   `width,height,depth`: Dimensions
    *   `r,g,b,a`: RGBA color values (0.0-1.0)
    *   `rot_x,rot_y,rot_z`: Rotation angles in degrees around X, Y, and Z axes
    *   `scale_x,scale_y,scale_z`: Scale factors along X, Y, and Z axes
*   `VAR;label;type;value` - Update a UI variable
    *   `label`: Variable name (should match element ID in UI)
    *   `type`: Variable type ("int", "float", "string")
    *   `value`: The variable value
*   `ARRAY;label;type;count;val1,val2,...` - Send an array of values
    *   `label`: Array name
    *   `type`: Array type ("int", "float")
    *   `count`: Number of elements
    *   `val1,val2,...`: Comma-separated values

### UI Variable Updates:

Variables sent from modules with matching labels to UI element IDs will automatically update those elements. This is particularly useful for updating text elements to display information based on canvas clicks or slider changes.

### Backward and Forward Compatibility:

*   Modules using the old 12-field SHAPE format continue to work unchanged (rotation defaults to 0, scale defaults to 1)
*   Modules using the new 18-field SHAPE format can specify rotation and scale transformations
*   The rendering system only applies transformations when rotation/scale values differ from defaults to maintain performance

## Dynamic DOM Matrix System

The CHTML framework now includes a simplified DOM matrix system that represents the UI as an indexable array of elements.

### Features:

*   **Simplified Implementation**: Uses the existing elements array as the "DOM matrix" instead of complex tree structures
*   **Element Querying**: Supports getElementById, getElementsByTagName, and querySelector-like functionality
*   **CSV Export**: Outputs the entire DOM structure to `dom_matrix.csv` for debugging
*   **Backward Compatibility**: Maintains all existing functionality

### Query Functions Available:

*   `get_element_index_by_id(id)` - Find element by ID
*   `get_element_index_by_tag_name(tag_name)` - Find element by tag name
*   `get_elements_by_tag_name(tag_name, results, max_results)` - Get all elements by tag name
*   `query_selector_all(selector, results, max_results)` - CSS-like selector functionality
*   `query_selector(selector)` - Get first element matching selector

### CSV Export Format:

The `dom_matrix.csv` file contains the following columns:
*   index: The array index of the element
*   type: The UI element type
*   id: The element ID
*   onClick: The onClick handler name
*   parent_index: The parent element's index (-1 if no parent)
*   x, y, width, height: Element position and size properties
