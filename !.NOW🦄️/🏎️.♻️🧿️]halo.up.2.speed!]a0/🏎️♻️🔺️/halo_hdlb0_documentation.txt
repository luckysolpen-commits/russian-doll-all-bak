# HALO:HDLB0 Hardware Description Language Documentation

## Overview
HALO:HDLB0 is a hardware description language and emulation system that allows users to design and simulate digital circuits using a binary instruction format. The system is implemented in C and provides an interactive environment for testing hardware designs.

## Core Components

### 1. HALO Emulator (0.hdlb0🧿️.☮️16]pr5]#ab]HALO.c)
The main C program that serves as the emulator for the HALO:HDLB0 system. It implements:
- A virtual machine that executes binary instructions representing hardware operations
- 256-byte RAM storage with persistence to ram_output_address.txt
- Interactive console interface with switches and clock control
- Support for external logic chip programs defined in chip_bank.txt
- Binary program parser for 16-bit instruction format

### 2. Instruction Format
Each instruction is 64 bits (4 words of 16 bits each):
- First 16 bits: chip_location (0 for direct logic, >0 for external chip calls)
- Second 16 bits: ram_output_address (where to store output, 0 for CLI tape)
- Third 16 bits: raw_input_a (first input parameter)
- Fourth 16 bits: raw_input_b (second input parameter)

### 3. Input Value Resolution
- 0: Logic 0
- 1: Logic 1
- 2 or 3: Blank/unused input
- 5: Switch 0 value
- 6: Switch 1 value
- 7: Current clock value
- 16-255: RAM addresses
- >255: RAM addresses (using modulo 256)

## How the System Works

The HALO:HDLB0 system operates in clock cycles:
1. Instructions are parsed from a program file in binary format
2. During each cycle, all instructions are executed in sequence
3. Inputs are resolved from switches, clock, RAM, or constants
4. Operations are either direct logic (chip_location = 0) or external chip programs
5. Outputs are written to RAM or to the CLI tape for display
6. The clock toggles between 0 and 1 after each cycle
7. RAM content is preserved between cycles in ram_output_address.txt

## Extending to Complex Chips: MS Flip-Flop

The system's architecture proves it can handle complex chips because:
1. It supports external logic chips via the chip_bank.txt mechanism
2. It provides RAM storage for state preservation
3. It has clock control for synchronous operations
4. It supports complex input/output operations
5. It can chain operations together in sequences

For an MS (Master-Slave) flip-flop, you would need:
- Two latches (master and slave)
- Clock inversion for the slave
- Proper state storage between clock cycles
- Edge-triggered behavior

## Prompt for Creating an MS Flip-Flop

### prompt_ms.txt
```
Create a Master-Slave flip-flop implementation for the HALO:HDLB0 hardware description language system.

CRITERIA for testable pass:
1. The flip-flop must have 3 inputs: Data (D), Clock (CLK), and Reset (RST)
2. The flip-flop must have 1 output: Q (and optionally Q_bar)
3. On the rising edge of the clock, the input D should be captured by the master latch
4. On the falling edge of the clock, the value from the master latch should be transferred to the slave latch and appear at output Q
5. When Reset is high, output Q should be forced to 0 regardless of clock or data input
6. The implementation should use two separate latches synchronized to opposite clock phases
7. The circuit should preserve state between clock cycles when not actively clocked
8. The implementation should be provided as either:
   - A binary program in HALO:HDLB0 format (sequence of 64-bit instructions)
   - C source code for an external chip program that can be added to chip_bank.txt

TECHNICAL SPECIFICATIONS:
- D input can be retrieved from switch_0 (value 5), switch_1 (value 6), or RAM address
- CLK input can be the system clock (value 7) or a RAM-stored clock signal
- RST input can be from switches or RAM
- Output Q should be stored in a specific RAM address for monitoring
- Use RAM addresses for internal state storage between clock phases
- The implementation should work within the 4-input instruction format of HALO:HDLB0

TEST SCENARIOS:
1. Verify that Q follows D on the appropriate clock edge
2. Verify that Q holds state between clock cycles
3. Verify that reset forces Q to 0 regardless of other inputs
4. Verify proper Master-Slave timing relationship (master samples on rising edge, slave updates on falling edge)
5. Verify the circuit works across multiple clock cycles with changing D inputs

The implementation should demonstrate that HALO:HDLB0 can be used to build sequential logic circuits with memory elements, proving its capability for complex digital system design.
```

This system proves that HALO:HDLB0 can form the basis for more complex digital circuits like flip-flops because it provides the fundamental building blocks: combinational logic, state storage (RAM), clock control, and input/output mechanisms. The proof program demonstrates these capabilities work correctly, enabling construction of more sophisticated sequential circuits.
