CHTM Layered Canvas Architecture - November 24 Update
=====================================================

ARCHITECTURE OVERVIEW:
- Layered rendering system replaces single canvas buffer approach
- Multiple discrete layers composed at render time
- Clean separation of concerns between different canvas elements

LAYER BREAKDOWN:

Layer 1: original_canvas[MAX_CANVAS_HEIGHT][MAX_CANVAS_WIDTH]
- Purpose: Static background elements that don't change often
- Content: Base terrain, walls, fixed objects
- Example: '.' (placeholder), '#' (walls), etc.
- Modification: Rare, usually at initialization

Layer 2: placed_objects[MAX_CANVAS_HEIGHT][MAX_CANVAS_WIDTH] 
- Purpose: Objects placed by user interaction (symbols from palette)
- Content: Symbols placed via palette selection, 'X' placements, etc.
- Example: '!', '$', '%', 'X', etc.
- Modification: Through character placement functions

Layer 3: player_layer[MAX_CANVAS_HEIGHT][MAX_CANVAS_WIDTH]
- Purpose: Player character positioning
- Content: Player 'p' character that moves around
- Example: 'p' (player), ' ' (empty)
- Modification: Through movement functions

COMPOSITION LOGIC:
In display_canvas() function:
for each position (r, c):
  1. Start with original_canvas[r][c]
  2. If placed_objects[r][c] != ' ', use that instead
  3. If player_layer[r][c] != ' ', use that instead (top priority)
  4. Send composite to renderer

KEY BENEFITS:
1. No more corruption from player/symbol conflicts
2. Clean separation of player movement vs object placement
3. Easy to debug individual layers
4. Predictable rendering behavior
5. Can place objects under player if needed

FUNCTION BEHAVIOR CHANGES:

- move_player(): Now only updates player_layer, no canvas modification
- place_selected_symbol(): Now only updates placed_objects layer  
- place_x_symbol(): New function to place 'X' in placed_objects layer
- display_canvas(): Composes all layers into single output for renderer

TESTING:
- Arrow keys: Move player around (affects only player_layer)
- X key: Place 'X' at player position (affects only placed_objects layer) 
- Enter key: Place selected palette symbol (affects only placed_objects layer)

This architecture solves the previous corruption issues by ensuring the player's 'p' 
character is always in its own dedicated layer, separate from the canvas modifications.
The composition happens at display time, ensuring clean, predictable output to the renderer.
