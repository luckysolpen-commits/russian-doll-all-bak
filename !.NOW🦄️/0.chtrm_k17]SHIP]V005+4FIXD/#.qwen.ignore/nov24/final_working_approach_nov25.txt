ANALYSIS OF WORKING EXAMPLES:
1. enhanced_game_ipc_v1.c: Sends ALL state after each command
2. model_v0.01.c: Parses structured messages like "SHAPE;x;y", "VAR;label;type;value"

BOTH EXAMPLES FOLLOW PATTERN:
- Receive command → Update state → Send complete current state → Flush

THE KEY PRINCIPLE:
Every command should result in a complete, synchronized update of both variables AND visual display data, sent together immediately.

CURRENT PROBLEM:
The renderer expects to get consistent data, but we're sending partial updates or variables without display data, causing synchronization issues.

WORKING SOLUTION PATTERN:
1. Each action updates state completely
2. Each action sends ALL relevant current variables (not just changed ones)
3. Each action may also send display data as needed
4. Each action ends with fflush to ensure immediate delivery

IMPLEMENTATION APPROACH:
For each action (move, place, etc.):
1. Update internal state
2. Send all relevant variables (frame count, player pos, etc.)
3. Send complete display if needed
4. Call fflush()

This ensures renderer always receives synchronized, complete state updates.
