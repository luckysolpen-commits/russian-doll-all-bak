# UI Texture Loading Fix Plan

## Problem
- Current texture loading happens during rendering in draw_element() function
- This blocks GUI response making the interface non-responsive
- Working backup version has GUI working but texture loading issues

## Solution Approach
- Move texture loading from rendering path to initialization/preloading step
- Use texture caching to avoid repeated loading
- Maintain all rendering functionality while improving GUI responsiveness

## Implementation Steps

1. **Analyze current working version (bak/3.view_v0.01_gui_on.c)**
   - Understand how it handles image loading without blocking GUI
   - Identify preloading mechanism

2. **Analyze broken version (3.view_v0.03]ui-broken.c)**
   - Identify where texture loading happens during render
   - Locate the image rendering code that causes blocking

3. **Create image preloading function**
   - Function to load all images referenced in UI elements before rendering
   - Use texture cache system to efficiently manage textures

4. **Modify element loading process**
   - When loading CHTML, identify image elements
   - Preload their textures before the rendering loop

5. **Update draw_element function**
   - Remove texture loading from rendering path
   - Use preloaded textures directly

6. **Maintain texture caching**
   - Keep efficient texture cache to avoid reloading
   - Handle texture cleanup properly

## Testing Plan
- Load CHTML files with image elements
- Verify GUI remains responsive
- Verify images still display correctly
- Test with multiple image types and sizes
