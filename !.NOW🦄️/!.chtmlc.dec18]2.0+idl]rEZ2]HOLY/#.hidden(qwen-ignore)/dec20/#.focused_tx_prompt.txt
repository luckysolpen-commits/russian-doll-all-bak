# Focused Prompt: Rotation Functionality with Backward Compatibility

## Goal
Add rotation and scaling capabilities to shapes while maintaining complete backward compatibility with existing modules.

## Requirements
1. Extend Shape structure with rotation (x, y, z) and scaling (x, y, z) fields
2. Update parsing to handle both old format (12 fields) and new format (18 fields) 
3. Default new fields to: rotations=0.0, scales=1.0 for backward compatibility
4. Update rendering functions to apply transformations
5. Create a simple self-rotating cube demo module

## Current Shape Format (12 fields):
"SHAPE;type;label;x;y;z;width;height;depth;r;g;b;a"

## Extended Shape Format (18 fields):
"SHAPE;type;label;x;y;z;width;height;depth;r;g;b;a;rot_x;rot_y;rot_z;scale_x;scale_y;scale_z"

## Implementation Steps
1. Update Shape struct in both model (2.model_v0.01.c) and view (3.view_v0.01.c)
2. Update parse_enhanced_format() to handle optional fields with defaults
3. Update both SHAPE parsing locations in update_model() function 
4. Update rendering functions to use transformation fields
5. Create simple self-rotating cube module (self_rotating_cube.c)
6. Create demo CHTML file (test_rotating_cube.chtml)

## Critical: Backward Compatibility
- When new fields are not present in SHAPE message, use defaults:
  - rotation_x=0.0, rotation_y=0.0, rotation_z=0.0  
  - scale_x=1.0, scale_y=1.0, scale_z=1.0
- Old modules should continue to work exactly as before
- New modules can optionally include rotation/scale fields
