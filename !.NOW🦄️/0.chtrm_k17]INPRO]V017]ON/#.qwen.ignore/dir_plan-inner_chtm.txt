# Directory Navigator Implementation Plan

## Overview
Plan for implementing a directory navigator element for file operations in the CHTM framework, with focus on HTML/JS innerHTML-like patterns.

## Goals
- Create a visual element that displays directories from the projects/ folder
- Allow navigation through directory structure
- Provide indexed selection capability
- Follow HTML/JS innerHTML patterns as closely as possible
- Separate presentation from logic (UI handles display, module handles operations)

## HTML/JS Comparison

### JavaScript innerHTML Pattern (Simple & Direct)
```javascript
// Simple, direct way to set dynamic content
const projectsContainer = document.getElementById('projects-browser');
const projects = getProjectsFromDirectory();

let htmlContent = '<h3>Available Projects:</h3>\n';
projects.forEach((project, index) => {
    htmlContent += `<button onclick="selectProject('${project}')">${project}</button>\n`;
});

projectsContainer.innerHTML = htmlContent;
```

This is exactly the pattern we should follow - simple, direct, and very HTML/JS-like.

## Approach Options

### Option 1: innerHTML-like Dynamic Content (Recommended)
**Implementation:**
- Module generates CHTM content string dynamically (like JavaScript creates HTML strings)
- Module sends content string to specific element ID (like innerHTML assignment)
- Renderer updates element content with new CHTM string (like innerHTML setting)
- Example: `INNER_HTML:projects_browser:<button key="1" onClick="SELECT_DIR:project1">project1</button>...`

**HTML/JS equivalent:**
```javascript
element.innerHTML = "<button onclick='...'>project1</button><button onclick='...'>project2</button>";
```

**Pros:**
- Most HTML/JS-like (directly mirrors innerHTML pattern)
- Simple and intuitive for developers
- Leverages existing CHTM syntax
- Easy to understand and implement
- Flexible for any content type

**Cons:**
- Requires renderer modification to handle dynamic content assignment
- Need to ensure proper element exists to update

### Option 2: Canvas with Dynamic Formatting (Simpler Implementation)
**Implementation:**
- Use existing canvas element to display formatted directory content
- Module generates formatted strings similar to CHTM but sent directly to canvas
- More like drawing content rather than DOM manipulation

**Pros:**
- Uses existing infrastructure
- Simpler to implement
- Maintains current architecture

**Cons:**
- Less HTML/JS-like
- More limited in functionality
- Not true element insertion

## Recommended Approach: Option 1 (innerHTML-like Dynamic Content)

This approach directly mirrors the JavaScript innerHTML pattern, which is one of the most common ways to dynamically update content in web development.

### Implementation Plan

#### 1. Element Definition
```
<div id="projects_browser">
  Loading projects...
</div>
```
*Note: Using a container element that can have its content replaced*

#### 2. Module Content Generation (JavaScript innerHTML-like)
```c
// Module generates CHTM content string dynamically - like JavaScript creates HTML strings
void update_project_list() {
    char content_buffer[4096] = {0};
    DIR* dir = opendir("projects/");
    struct dirent* entry;
    
    strcat(content_buffer, "<h3>Available Projects:</h3>\n");
    
    int index = 1;
    while ((entry = readdir(dir)) != NULL) {
        if (entry->d_type == DT_DIR && 
            strcmp(entry->d_name, ".") != 0 && 
            strcmp(entry->d_name, "..") != 0) {
            
            char item[256];
            snprintf(item, sizeof(item), 
                "<button key=\"%d\" onClick=\"SELECT_DIR:%s\">%s</button><br/>\n", 
                index, entry->d_name, entry->d_name);
            strcat(content_buffer, item);
            index++;
        }
    }
    
    // Send to renderer to update element content (like JavaScript element.innerHTML = content)
    printf("INNER_HTML:projects_browser:%s\n", content_buffer);
    fflush(stdout);
}
```

#### 3. JavaScript Equivalent
```javascript
// What we're modeling after:
const projectsContainer = document.getElementById("projects_browser");
let content = '<h3>Available Projects:</h3>\n';

projects.forEach((project, index) => {
    content += `<button onclick="selectProject('${project}')">${project}</button><br/>\n`;
});

projectsContainer.innerHTML = content;
```

#### 4. Communication Protocol
- `INNER_HTML:element_id:chtm_content` - Replace content of element with ID (exactly like JavaScript innerHTML)
- Module sends this command after scanning directories
- Renderer finds existing element and replaces its content

#### 5. Interaction Flow
```
1. Module scans projects directory  
2. Module generates CHTM string dynamically with buttons for each project
3. Module sends INNER_HTML command to update specific element
4. Renderer replaces content of element (like JavaScript innerHTML)
5. User interacts with new elements (buttons, links, etc.)
6. Events are processed normally through existing onClick system
```

## Technical Implementation Steps

### Phase 1: Renderer Command Processing
1. Add INNER_HTML command parsing to main input processing
2. Implement element content replacement logic
3. Validate element exists before updating

### Phase 2: Module Integration  
1. Create directory scanning and CHTM generation functions
2. Send INNER_HTML commands when directory content needs updating
3. Handle button selection events from dynamically created elements

### Phase 3: Testing and Refinement
1. Test dynamic content replacement
2. Verify interactive elements work properly
3. Ensure visual feedback works with dynamic elements

## Advantages of This Approach
- Directly mimics JavaScript innerHTML pattern (most HTML/JS-like)
- Simple and intuitive concept
- Maximum flexibility for content generation
- Clear separation: module generates content like JS, renderer displays like HTML
- Extensible to any dynamic content scenario
- Familiar pattern for JavaScript developers

## Considerations
- Requires renderer modification to handle the new command
- Need to ensure security (validate content being inserted)
- Must handle existing event binding patterns with dynamic content
- Element must exist before trying to update its content
