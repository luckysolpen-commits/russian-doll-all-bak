CHTM Framework Menu Navigation Bug Fix

## Current State
We have a CHTM (Custom HTML for Terminal Menu) framework that renders CLI interfaces with an MVC-like architecture. Menu elements have child items that should be navigable when the parent menu is open.

## Problem Description
When a user navigates to a menu element and presses Enter to activate it, the following issue occurs:
- Menu children disappear from the display
- Navigation breaks and no elements can be selected
- The menu closes or becomes inactive unexpectedly

## Expected Behavior
When a menu is activated with Enter:
1. The menu remains active [^] and visible (this  is currently working fine , its the children that dont activate properly with [^]
2. Child elements appear under the menu as "Menu Items:"
3. Child elements should be navigable with arrow keys [>] 
4. Pressing Enter on a child element makes it "active" [^]; waits to executes its action until enter is pressed once more after being active
5. ESC key should return focus to parent menu 

## Current Menu Structure Example
```
Current file: chtm/menu_test.chtm

Menu System Test
[^] 1. Main Menu:
  Menu Items:
    [ ] Option 1
    [ ] Option 2
    [^] Option 3  # This should be possible after activation
    
Status: File Operations Ready
[ ] 2. [Open Menu][ ] 3. [Back to Main][ ] 4. Command: ${command_input}
```

## What Currently Happens
Instead, when Enter is pressed on a menu:
```
Current file: chtm/menu_test.chtm

Menu System Test
[ ] 1. Main Menu:  # Menu becomes inactive and children disappear

Status: File Operations Ready
[ ] 2. [Open Menu][ ] 3. [Back to Main][ ] 4. Command: ${command_input}
```

## Relevant Code Context
- active_interaction_element: tracks which element is currently active [^]
- focused_element_index: tracks which element has navigation focus [>]
- is_element_visible(): determines if an element should render
- is_interactive_element(): determines if an element can be navigated to
- UIElement.parent_index: -1 for top-level, index of parent for children
- UIElement.children[]: array of child element indices
- UIElement.current_child_focus: currently focused child in active menu

## Desired Fix
Ensure that menu activation properly maintains state so children remain visible and navigation continues to work properly after menu activation.
