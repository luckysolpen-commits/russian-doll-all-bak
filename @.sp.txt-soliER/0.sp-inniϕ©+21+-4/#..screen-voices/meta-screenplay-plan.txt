## OVERVIEW

Convert ALL screenplay scene files (~345 scenes across 15+ books) to multi-voice audio using edge-tts. These are our most valuable assets for animation. Each scene file becomes an MP3 with distinct character voices + narrator.

---

## FORMAT FAMILIES

Screenplays use TWO format families. We need parsers for BOTH:

### Family A — HTML/Blockquote (Books: Sol Pen, Geminii, Red Rock, Moonshine, SUPERWAR, Hellride, AVANTA.C, Digi_Sol, Metal Sun, XO Files, Footrace.FU, Centroid)

```
<center>LUCKY</center>
> (Frantic)
> Astra! What is this place?!

<center>ASTRA</center>
> You orchestrated this?
```

**Parser Logic:**
1. Detect `<center>NAME</center>` → switch to character voice
2. Read subsequent `> ` lines as dialogue (skip `> (parenthetical)` lines)
3. Non-`>` lines → narrator (action/description)
4. `**...**` lines → narrator (scene headings)

### Family B — Plain Text (Books: Death-Kryo!, CHAR.A.NON, NGL Mpyre, Tear-It)

```
LUCKY
(grinning)
Thank you! Thank you all for coming.

LUCKY (CONT'D)
But this isn't just my victory.

UN AGENT
(a synthesized voice)
Lucky is guilty.
```

**Parser Logic:**
1. Detect ALL CAPS line (optionally with `(CONT'D)`, `(age...)`, or `V.O.` prefix) → switch to character voice
2. Read subsequent plain text lines as dialogue until next ALL CAPS name or blank line
3. Skip `(parenthetical)` lines
4. Prose paragraphs → narrator
5. Edge cases: `NAME (50s), his hair...` → narrator (character intro, not dialogue)

## CONVERSION STRATEGY

### Phase 1: Build the Parser
Create `screenplay_tts.py` — a universal screenplay parser that:
1. Auto-detects Family A vs Family B format
2. Parses character names, dialogue, parentheticals, action lines
3. Maps characters to voices using the voice map above
4. Generates segment MP3s and concatenates with ffmpeg
5. Handles edge cases: character intros, SOUND cues, FLASHBACK blocks, CONT'D, V.O.

### Phase 2: Per-Book Voice Maps
Each book has its own character roster. Create a voice map file per book:
- `screen-voices/voice-map-sol-pen.txt`
- `screen-voices/voice-map-geminii.txt`
- `screen-voices/voice-map-death-kryo.txt`
- etc.

### Phase 3: Convert Book by Book
Start with Sol Pen (31 scenes), then work through the list. Each book gets:
- Its own converter script (or one universal script with book-specific voice map)
- Output directory: `screen-voices/{book-name}/`
- Output naming: `{book}.scene{N}.mp3`

### Phase 4: Quality Check
- Spot-check 2-3 scenes per book
- Verify character voices are distinct
- Verify narrator voice is consistent
- Verify parentheticals are skipped
- Verify scene headings are read by narrator

## OUTPUT STRUCTURE

**MP3s go in an `audio/` subfolder INSIDE the original screenplay directory:**
```
SP.all-writeez28-b2/{book-dir}/{scene-dir}/
├── scene_01.txt
├── scene_02.txt
├── ...
└── audio/                          ← MP3s go HERE
    ├── {book-shortname}-scene-01.mp3
```

## EDGE CASES TO HANDLE

| Edge Case | Example | Treatment |
|-----------|---------|-----------|
| Character intro | `LUCKY SHEPHERD (50s), his hair now streaked...` | Narrator (prose, not dialogue) |
| CONT'D | `LUCKY (CONT'D)` | Same character voice |
| V.O. | `V.O. LUCKY` | Narrator or ethereal voice |
| SOUND cues | `SOUND of slot machines` | Narrator |
| FLASHBACK blocks | `FLASHBACK ... END FLASHBACK` | Narrator |
| CLOSE UP | `CLOSE UP - Lucky's face` | Narrator |
| FADE OUT | `FADE TO BLACK.` | Skip |
| Generic names | `MERC 1`, `UN AGENT` | Valid speakers, assign from NPC pool |
| Multiple speakers in one line | Rare | Split by context |
| Empty dialogue | `LUCKY\n\n` | Skip |
