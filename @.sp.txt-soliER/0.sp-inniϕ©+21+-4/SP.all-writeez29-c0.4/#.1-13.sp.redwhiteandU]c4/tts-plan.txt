## Single-Voice Book Conversion (edge-tts)

**Script:** `batch_convert_book0_edge_tts.py`
**Voice:** `en-US-JennyNeural` (warm, friendly narrator)
**Output naming:** `sp.chapter{N}-v1.0.mp3`

### How to convert a new book:

1. **Copy the script:**
   ```bash
   cp batch_convert_book0_edge_tts.py batch_convert_bookX_edge_tts.py
   ```

2. **Edit the BOOK_DIR and OUTPUT_PREFIX** in the script:
   ```python
   BOOK_DIR = "path/to/book/directory"
   OUTPUT_PREFIX = "bookshortname.chapter"  # e.g. "rr.chapter" for Red Rock
   ```

3. **Run it:**
   ```bash
   python3 batch_convert_bookX_edge_tts.py
   ```

4. **It auto-skips** existing MP3 files. Safe to re-run.

## Screenplay/Script Format (Multi-Voice)

### Format Example: `scene_01_casino_flashback.txt`
Standard screenplay format:
```
INT. LUXURY CASINO RESORT - DAY (FLASHBACK)

SOUND of slot machines, distant laughter

LUCKY SHEPHERD (30s), sharp-suited and radiating charm...

A MONTAGE begins, quick cuts...

FADE TO BLACK.
```

### Key Elements:
Screenplays use **TWO format families**:

**Family A — HTML/Blockquote (Books 0, 1, 2):**
```
<center>LUCKY</center>
> (Frantic)
> Astra! What is this place?!

<center>ASTRA</center>
> You orchestrated this?
```

**Family B — Plain Text (Books 3, 5, 111):**
```
LUCKY
(cautious)
What do you want, Dorchester?

DORCHESTER (CONT'D)
These are just puppets.
```

| Element | Family A Pattern | Family B Pattern | TTS Treatment |
|---------|-----------------|-----------------|---------------|
| **Character name** | `<center>NAME</center>` | `NAME` (ALL CAPS line) | Switch to character voice |
| **Dialogue** | `> text` | plain text line after name | Character voice |
| **Parenthetical** | `> (direction)` | `(direction)` on own line | Skip (direction only) |
| **(CONT'D)** | N/A | `NAME (CONT'D)` | Same character voice |
| **V.O.** | N/A | `V.O. NAME` | Narrator or ethereal voice |
| **Scene heading** | `**INT. LOC - TIME**` or plain | same | Narrator voice |
| **Action/description** | prose paragraphs | prose paragraphs | Narrator voice |
| **SOUND cues** | `SOUND of ...` | `SOUND of ...` | Narrator voice |
| **Transitions** | `FADE OUT.`, `FADE TO BLACK.` | same | Skip |
| **CLOSE UP** | N/A | `CLOSE UP - ...` | Narrator voice |
| **FLASHBACK blocks** | N/A | `FLASHBACK ... END FLASHBACK` | Narrator voice |

### Parsing Challenge:
Screenplay format is **NOT dialogue-labeled** like D&D sessions. Need a screenplay-aware parser that handles **both format families**:

**Family A parser logic:**
1. Detect `<center>NAME</center>` → switch to character voice
2. Read subsequent `> ` lines as dialogue (skip `> (parenthetical)` lines)
3. Non-`>` lines → narrator (action/description)

**Family B parser logic:**
1. Detect ALL CAPS line (optionally with `(CONT'D)`, `(age...)`, or `V.O.` prefix) → switch to character voice
2. Read subsequent plain text lines as dialogue until next ALL CAPS name or blank line
3. Skip `(parenthetical)` lines
4. Prose paragraphs → narrator

**Edge cases to handle:**
- Character intros: `LUCKY SHEPHERD (50s), his hair now streaked...` — ALL CAPS name but followed by comma + prose, NOT dialogue → narrator
- `SOUND of ...` lines → narrator, not dialogue
- `CLOSE UP - ...` → narrator
- `FLASHBACK ... END FLASHBACK` blocks → narrator
- Generic names: `MERC 1`, `UN AGENT`, `BORIN IRONMUG` → valid speakers

## YouTube Video Pipeline (Audio + Image Slides)

### Concept
Each D&D session audio gets paired with a slideshow of AI-generated images:
- **Player/character portraits** — unique "home base" image per character
- **Environment/scene images** — locations (tavern, temple, jungle, etc.)
- **Enemy/NPC images** — bosses, monsters, key NPCs
- Images change at natural scene/segment transitions

### Image Generation Options

| Option | API/Tool | Cost | Quality | Notes |
|--------|----------|------|---------|-------|
| **Stable Diffusion** (local) | `diffusers` + SDXL | Free (GPU needed) | High | Needs good GPU, full control |
| **DALL-E 3** | OpenAI API | ~$0.04-0.08/image | Very High | Best prompt understanding |
| **Flux** (local/API) | `diffusers` or Replicate | Free/$ | Very High | Newer, excellent quality |
| **Midjourney** | Discord bot | $10/mo | Very High | No API, manual only |

### Recommended: **Flux (local, free)** — deferred
- DALL-E 3 and Replicate require API keys/payment — not available now
- **Plan: run Flux or SD locally on a separate computer later**
- For now: audio-only output. Image/video pipeline on hold.
